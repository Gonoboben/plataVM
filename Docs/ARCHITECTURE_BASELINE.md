# Архитектурная база PlataVM

Дата фиксации: 2026-06-30

Документ фиксирует начальную архитектурную базу проекта PlataVM / плата питания.

## 1. Граница архитектуры

Репозиторий:

```text
Gonoboben/plataVM
```

Проект:

```text
PlataVM / плата питания / энергетический модуль БНПА
```

## 2. Назначение

PlataVM предназначен для разработки платы питания и распределения энергии в составе подводного буксируемого комплекса / БНПА.

Основные функции:

1. приём питания от аккумуляторных блоков;
2. распределение питания по нагрузкам;
3. управление силовыми каналами;
4. защита силовых цепей;
5. резервное питание критических узлов;
6. телеметрия состояния питания;
7. связь с главным управляющим модулем.

## 3. Исходные данные

На дату фиксации приняты следующие предварительные параметры:

1. аккумуляторная химия: `LiFePO4`;
2. номинальное напряжение батарейной сборки: `12,8 В`;
3. основная батарейная архитектура: `4S`;
4. основная шина: `BAT_BUS_4S`;
5. осветители: `6 шт × 30 Вт = 180 Вт`;
6. прочая электроника: ориентировочно `40 Вт`;
7. суммарная основная нагрузка: ориентировочно `220 Вт`.

Эти данные требуют дальнейшей детализации по расчётам, схемам и BOM.

## 4. Предварительная структура питания

Базовая логика основного питания:

```text
BAT1 + BAT2 → параллельно → BAT_BUS_4S → силовые DC/DC и ключи → нагрузки
```

Базовая логика резервного питания:

```text
EMG_4S2P → CRITICAL_INPUT → 5V_CRIT / 3V3_CRIT → критические узлы
```

Ограничение:

```text
EMG_4S2P не питает ИЗС и BOOST 32 В
```

## 5. Группы нагрузок

Предварительные группы:

1. ИЗС / осветители;
2. видеосистема;
3. датчики;
4. связь / телеметрия;
5. управляющая логика;
6. критическое питание;
7. сервисные интерфейсы.

Для каждой группы должны быть отдельно зафиксированы:

1. номинальное напряжение;
2. максимальный ток;
3. пусковой ток;
4. защита;
5. допустимый режим отключения;
6. принадлежность к основной или критической шине.

## 6. Интерфейсы

Основной интерфейс связи с главным модулем:

```text
RS-485
```

Сервисные интерфейсы:

```text
UART / SWD
```

Любая замена основного интерфейса требует ADR.

## 7. Земля и помехозащита

Базовое правило:

```text
Power GND и Signal GND не объединяются произвольно.
```

Для соединения земель должна быть явно определена точка или схема:

```text
STAR POINT / контролируемая точка соединения / обоснованная схема земли
```

## 8. Защиты

Минимально должны быть предусмотрены и рассчитаны:

1. защита от переполюсовки;
2. защита от короткого замыкания;
3. защита каналов нагрузки;
4. защита от перегрузки;
5. контроль напряжения основной шины;
6. контроль критической шины;
7. контроль температуры силовых элементов;
8. безопасное отключение мощных нагрузок.

## 9. Архитектурные ограничения

Без отдельного ADR нельзя:

1. подключать резервную шину к осветителям;
2. питать BOOST 32 В от резервного источника;
3. менять BAT_BUS_4S на другую архитектуру;
4. объединять земли без схемы и проверки;
5. менять RS-485 как основной интерфейс.

## 10. Открытые вопросы

Требуют уточнения:

1. точный состав нагрузок;
2. номиналы всех шин;
3. максимальные токи по каналам;
4. пусковые токи осветителей и преобразователей;
5. типы силовых ключей;
6. типы DC/DC-преобразователей;
7. схема зарядки/обслуживания аккумуляторов;
8. способ измерения токов;
9. требования к герметизации и разъёмам;
10. температурный диапазон;
11. требования к испытаниям.

## 11. Правило изменения этой базы

Этот файл можно менять только с явным описанием:

1. какая часть архитектуры меняется;
2. почему прежняя архитектура недостаточна;
3. какие электрические, тепловые и эксплуатационные риски появляются;
4. какие проверки должны подтвердить корректность;
5. какие прежние решения сохраняются.
