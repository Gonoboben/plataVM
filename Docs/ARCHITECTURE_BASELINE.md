# Архитектурная база ПДУ БНПА / PlataVM

Статус:

```text
V1.5 corrected baseline — bistable battery contactors
```

## 1. Назначение

Файл фиксирует базовую архитектуру питания, управления, резервирования и выходных каналов.

## 3. Базовые шины

| Шина | Источник | Назначение |
|---|---|---|
| PACK_BUS | АКБ_1/АКБ_2 через Battery Front-End | главная силовая шина аппарата |
| POWER_12V_BUS | PACK_BUS | 14 внешних 12 В каналов |
| 5V_SYS_BUS | PACK_BUS через отдельный DC/DC 12→5 В | 10 внешних 5 В выходов |
| LIGHT_POWER_BRANCH | PACK_BUS | питание световых матриц через собственные LED-драйверы |
| RESERVE_BRANCH | PACK_BUS + EMG | заряд EMG и critical power path |
| 5V_CRIT | critical power path | MCU, связь и аварийная логика |
| 3V3_CRIT | 5V_CRIT | цифровая логика |

Отдельные `MAIN_INPUT_BUS` и центральный `K_MAIN` отсутствуют.

## 4. Батарейная архитектура

```text
АКБ_1 → BMS → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                       ├→ PACK_BUS
АКБ_2 → BMS → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘
```

Внутри каждого корпуса АКБ находятся только:

1. аккумуляторная сборка;
2. существующая BMS;
3. предохранитель;
4. однополюсный силовой контактор в положительном проводнике;
5. необходимая проводка;
6. минимальные пассивные элементы защиты катушки контактора.

На корпусе каждой АКБ установлены:

1. `СН-176А-12` для `BAT+`, `BAT−` и `REMOTE_OFF`;
2. отдельный двухконтактный разъём `LOCAL_START`.

В корпусе АКБ отсутствуют отдельный MCU, дополнительная управляющая плата, RS-485, CAN, UART, цифровая телеметрия BMS и дополнительный DC/DC для управляющей электроники.

В корпусе электроники находятся измерения батарейных ветвей, электронные силовые тракты Battery Front-End, ограниченные тракты `DECK_BALANCE`, диагностика, аппаратный `EXT_KILL` и формирователи импульсов `REMOTE_OFF`.

## 5. Управление K_BAT1/K_BAT2

Контакторы работают импульсно и бистабильно:

```text
кратковременный LOCAL_START → силовая цепь замкнута
кратковременный REMOTE_OFF  → силовая цепь разомкнута
```

После окончания импульса контактор сохраняет выбранное состояние. Постоянное питание катушки не требуется и не допускается как рабочая архитектура.

Конкретная конструкция контактора пока не выбрана. Допускаются:

1. две катушки SET/RESET;
2. одна реверсивная катушка.

В обоих случаях должны сохраняться внешние функции `LOCAL_START` и `REMOTE_OFF`.

Моностабильный контактор исключён из текущей рабочей архитектуры.

## 6. Интерфейсы батарейной линии

Рабочая распайка `СН-176А-12`:

| Контакты | Назначение |
|---|---|
| 1–5 | BAT+ |
| 6–10 | BAT− |
| 11 | REMOTE_OFF, кратковременный импульс выключения |
| 12 | RESERVE, пока не назначен |

Через `СН-176А-12` не передаются RS-485, CAN, UART или цифровая телеметрия BMS.

`LOCAL_START` передаётся только через отдельный двухконтактный разъём и не проходит через `СН-176А-12`.

## 7. Режимы аккумуляторной подсистемы

### RUN

Обе основные АКБ работают параллельно. Различия напряжений, токов и расчётного SoC используются для контроля и диагностики состояния батарейных ветвей, но сами по себе не являются причиной автоматического отключения исправной батареи.

### DECK_BALANCE

На палубе тяжёлые нагрузки отключены, а MCU управляет контролируемым межблочным током через ограниченный тракт в корпусе электроники.

### HARD_OFF

Штатное физическое отключение выполняется кратковременными импульсами `REMOTE_OFF` для `K_BAT1` и `K_BAT2` после отключения тяжёлых нагрузок и сохранения журнала.

Аварийное отключение выполняется аппаратной цепью `EXT_KILL`, которая независимо от MCU формирует импульсы `REMOTE_OFF` для обоих контакторов и отключает электронные силовые тракты Battery Front-End.

## 8. Ограничения

1. EMG не питает POWER_12V_BUS.
2. EMG не питает 5V_SYS_BUS.
3. EMG не питает LIGHT_POWER_BRANCH.
4. POWER_12V_BUS не питает LIGHT_POWER_BRANCH.
5. 5V_SYS_BUS не питает MCU critical domain.
6. TRACO не является целевой архитектурой новой платы.
7. CAN/CAN-FD не является baseline.
8. Межблочное выравнивание выполняется только с ограничением тока.
9. Нельзя полагаться на цифровую телеметрию BMS основных АКБ.
10. Нельзя использовать батарейный соединитель для дополнительной цифровой связи без нового ADR.
11. Нельзя удерживать `LOCAL_START` постоянно замкнутым.
12. Нельзя удерживать `REMOTE_OFF` как постоянный управляющий уровень.
13. Нельзя выбирать конкретную модель контактора до анализа электрической реализации импульсов и требований к контактору.
14. Нельзя вводить новые функции и дополнительную электронику до закрытия текущих вопросов.

## 9. Схемотехнический рубеж

Следующий блокирующий этап:

1. определить электрическую реализацию `LOCAL_START`;
2. определить электрическую реализацию `REMOTE_OFF`;
3. сформировать требования к бистабильному контактору;
4. только после этого выбирать конкретную модель контактора.
