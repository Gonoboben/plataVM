Статус:

```text
V1.5 integrated TO-BE baseline — battery architecture clarification
```

## 1. Назначение

Файл фиксирует базовую архитектуру питания, управления, резервирования и выходных каналов.

## 3. Базовые шины

| Шина | Источник | Назначение |
|---|---|---|
| PACK_BUS | АКБ_1/АКБ_2 через Battery Front-End | главная силовая шина аппарата |
| POWER_12V_BUS | PACK_BUS | 14 внешних 12 В каналов |
| 5V_SYS_BUS | PACK_BUS через отдельный DC/DC 12→5 В | 10 внешних 5 В выходов |
| LIGHT_POWER_BRANCH | PACK_BUS | питание световых матриц |
| RESERVE_BRANCH | PACK_BUS + EMG | заряд EMG и critical power path |
| 5V_CRIT | critical power path | MCU, связь и аварийная логика |
| 3V3_CRIT | 5V_CRIT | цифровая логика |

Отдельные `MAIN_INPUT_BUS` и центральный `K_MAIN` не используются.

## 4. Батарейная архитектура

```text
АКБ_1 → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                      ├→ PACK_BUS
АКБ_2 → K_BAT2 → СН-176А-12 → BFE_2 ┘
```

Каждый Battery Front-End содержит измерения, `PACK_PRESENT`, основной электронный силовой тракт, ограниченный балансировочный тракт и быстродействующую защиту.

## 5. Режимы аккумуляторной подсистемы

### RUN

Обе исправные АКБ работают параллельно. Разница напряжений, токов и SoC контролируется и отображается, но сама по себе не является причиной отключения батареи.

### DECK_BALANCE

На палубе тяжёлые нагрузки отключены, а MCU управляет контролируемым межблочным балансировочным током между АКБ_1 и АКБ_2.

Отдельная функция `BATTERY_MISMATCH` отсутствует.

### HARD_OFF

Физическое отключение основных источников выполняется командами OFF на `K_BAT1` и `K_BAT2`. При штатном завершении журнал сохраняется до размыкания контакторов; при аварийном завершении код причины сохраняется после отключения основных АКБ за счёт EMG.

## 6. Ограничения

1. EMG не питает POWER_12V_BUS.
2. EMG не питает 5V_SYS_BUS.
3. EMG не питает LIGHT_POWER_BRANCH.
4. POWER_12V_BUS не питает LIGHT_POWER_BRANCH.
5. 5V_SYS_BUS не питает MCU critical domain.
6. TRACO не является целевой архитектурой новой платы.
7. CAN/CAN-FD не является baseline.
8. Межблочное выравнивание выполняется только с ограничением тока.
