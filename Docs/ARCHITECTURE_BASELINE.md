# Архитектурная база ПДУ БНПА / PlataVM

Статус:

```text
V1.5 integrated TO-BE baseline — minimal battery housing
```

## 1. Назначение

Файл фиксирует базовую архитектуру питания, управления, резервирования и выходных каналов.

## 3. Базовые шины

| Шина | Источник | Назначение |
|---|---|---|
| PACK_BUS | АКБ_1/АКБ_2 через Battery Front-End | главная силовая шина аппарата |
| POWER_12V_BUS | PACK_BUS | 14 внешних 12 В каналов |
| 5V_SYS_BUS | PACK_BUS через отдельный DC/DC 12→5 В | 10 внешних 5 В выходов |
| LIGHT_POWER_BRANCH | PACK_BUS | питание световых матриц |
| RESERVE_BRANCH | PACK_BUS + EMG | заряд EMG и critical power path |
| 5V_CRIT | critical power path | MCU, связь и аварийная логика |
| 3V3_CRIT | 5V_CRIT | цифровая логика |

Отдельные `MAIN_INPUT_BUS` и центральный `K_MAIN` не используются.

## 4. Батарейная архитектура

```text
АКБ_1 → BMS → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                     ├→ PACK_BUS
АКБ_2 → BMS → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘
```

В корпусе каждой АКБ остаются только сборка, BMS, предохранитель, K_BATx, пассивное подавление катушки и проводка.

В корпусе электроники находятся:

1. измерения BATx_V/BATx_I;
2. драйверы K_BAT1/K_BAT2;
3. MAIN_SW1/MAIN_SW2;
4. BALANCE_SW1/BALANCE_SW2 и R_BAL1/R_BAL2;
5. защита и диагностика.

## 5. Интерфейсы батарейной линии

Через СН-176А-12 не передаются RS-485, CAN, UART или цифровая телеметрия BMS.

Контакты 11–12 используются только для управления контактором и возможной пассивной обратной связи.

## 6. Режимы аккумуляторной подсистемы

### RUN

Обе исправные АКБ работают параллельно. Различие напряжений, токов и SoC контролируется, но не является причиной автоматического отключения исправной батареи.

### DECK_BALANCE

На палубе тяжёлые нагрузки отключены, а MCU управляет контролируемым межблочным током через ограниченный тракт в корпусе электроники.

### HARD_OFF

Физическое отключение основных источников выполняется снятием управления K_BAT1 и K_BAT2. HARD_OFF не зависит от цифровой связи с аккумуляторными корпусами.

## 7. Ограничения

1. EMG не питает POWER_12V_BUS.
2. EMG не питает 5V_SYS_BUS.
3. EMG не питает LIGHT_POWER_BRANCH.
4. POWER_12V_BUS не питает LIGHT_POWER_BRANCH.
5. 5V_SYS_BUS не питает MCU critical domain.
6. TRACO не является целевой архитектурой новой платы.
7. CAN/CAN-FD не является baseline.
8. Межблочное выравнивание выполняется только с ограничением тока.
9. Нельзя полагаться на цифровую телеметрию BMS основных АКБ.
10. Нельзя использовать батарейный соединитель для дополнительной цифровой связи без нового ADR.
