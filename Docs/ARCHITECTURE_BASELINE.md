Статус:

```text

```

Файл фиксирует базовую архитектуру питания, управления, резервирования и выходных каналов.

## 3. Базовые шины

| Шина | Источник | Назначение |
|---|---|---|
| PACK_BUS | АКБ_1/АКБ_2 через Battery Front-End | главная силовая шина аппарата |
| POWER_12V_BUS | PACK_BUS | 14 внешних 12 В каналов |
| 5V_SYS_BUS | PACK_BUS через отдельный DC/DC 12→5 В | 10 внешних 5 В выходов |
| LIGHT_POWER_BRANCH | PACK_BUS | питание световых матриц через собственные LED-драйверы |
| RESERVE_BRANCH | PACK_BUS + EMG | заряд EMG и critical power path |

| 3V3_CRIT | 5V_CRIT | цифровая логика |

Отдельные `MAIN_INPUT_BUS` и центральный `K_MAIN` отсутствуют.

## 4. Батарейная архитектура

```text
АКБ_1 → autonomous BMS → protected output → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                                                       ├→ PACK_BUS
АКБ_2 → autonomous BMS → protected output → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘
```

Внутри каждого корпуса АКБ находятся только:

1. аккумуляторная сборка;
2. существующая автономная BMS;
3. предохранитель;
4. однополюсный моностабильный силовой контактор в положительном проводнике;
5. одна катушка непрерывного режима;
6. механически связанный вспомогательный NO-контакт самоподхвата;
7. необходимая проводка;
8. минимальные пассивные элементы защиты катушки.

На корпусе каждой АКБ установлены:

1. `СН-176А-12` для `BAT+`, `BAT−` и `REMOTE_OFF`;
2. отдельный двухконтактный разъём `LOCAL_START`.

В корпусе АКБ отсутствуют отдельный MCU, дополнительная управляющая плата, RS-485, CAN, UART, цифровая телеметрия BMS и дополнительный DC/DC для управляющей электроники.

Проект не использует raw-ветвь, внутренние точки BMS, дискретные сигналы BMS или управление её силовыми ключами. Тип коммутации BMS не влияет на схему контактора.

В корпусе электроники находятся измерения батарейных ветвей, электронные силовые тракты Battery Front-End, ограниченные тракты `DECK_BALANCE`, диагностика, аппаратный `EXT_KILL` и элементы разрыва цепей `REMOTE_OFF`.

## 5. Управление K_BAT1/K_BAT2

### 5.1 Функциональная схема

```text
BAT_PROT+ ─ F_CTRL ─┬─ LOCAL_START_NO ─┐
                    └─ K_BAT_AUX_NO ───┤─ K_BAT_COIL ─ pin 11 REMOTE_OFF
                                                        │
                                                        ├─ EXT_KILL_NC
                                                        ├─ REMOTE_OFF_NC
BAT_PROT− ──────────────────────────────────────────────┘
```

### 5.2 Включение

```text
кратковременно замкнуть LOCAL_START
→ катушка получает питание
→ главный контакт K_BATx закрывается
→ K_BAT_AUX_NO замыкается
→ после отпускания LOCAL_START катушка удерживается через K_BAT_AUX_NO
```

### 5.3 Выключение

```text
разомкнуть REMOTE_OFF/EXT_KILL loop
→ катушка обесточивается
→ K_BATx пружиной возвращается в OPEN
→ K_BAT_AUX_NO размыкается
→ восстановление loop не вызывает повторного включения
```

### 5.4 Потеря выхода BMS

```text
BAT_PROT+ или BAT_PROT− исчезает
→ катушка обесточивается
→ K_BATx автоматически OPEN
→ выход BMS восстанавливается
→ K_BATx остаётся OPEN до нового LOCAL_START
```

## 6. Интерфейсы батарейной линии

Рабочая распайка `СН-176А-12`:

| Контакты | Назначение |
|---|---|
| 1–5 | BAT+ |
| 6–10 | BAT− |
| 11 | REMOTE_OFF / K_BAT_HOLD_RETURN, участок возвратной цепи катушки |
| 12 | RESERVE, пока не назначен |

Через `СН-176А-12` не передаются RS-485, CAN, UART или цифровая телеметрия BMS.

`LOCAL_START` передаётся только через отдельный двухконтактный разъём и не проходит через `СН-176А-12`.

## 7. Режимы аккумуляторной подсистемы

### RUN

Обе основные АКБ работают параллельно. Различия напряжений, токов и расчётного SoC используются для контроля и диагностики состояния батарейных ветвей, но сами по себе не являются причиной автоматического отключения исправной батареи.

### DECK_BALANCE

На палубе тяжёлые нагрузки отключены, а MCU управляет контролируемым межблочным током через ограниченный тракт в корпусе электроники.

### HARD_OFF

Штатное физическое отключение выполняется размыканием цепей удержания `K_BAT1` и `K_BAT2` после отключения тяжёлых нагрузок и сохранения журнала.

Аварийное отключение выполняется аппаратной цепью `EXT_KILL`, которая независимо от MCU размыкает обе цепи удержания и отключает электронные силовые тракты Battery Front-End.

## 8. Ограничения

1. EMG не питает POWER_12V_BUS.
2. EMG не питает 5V_SYS_BUS.
3. EMG не питает LIGHT_POWER_BRANCH.
4. POWER_12V_BUS не питает LIGHT_POWER_BRANCH.
5. 5V_SYS_BUS не питает MCU critical domain.
6. TRACO не является целевой архитектурой новой платы.
7. CAN/CAN-FD не является baseline.
8. Межблочное выравнивание выполняется только с ограничением тока.
9. Нельзя полагаться на цифровую телеметрию BMS основных АКБ.
10. Нельзя использовать батарейный соединитель для дополнительной цифровой связи без нового ADR.
11. Нельзя использовать raw-ветвь или внутренние точки BMS для LOCAL_START/REMOTE_OFF.
12. `LOCAL_START` должен быть нормально разомкнутым, пружинно-возвратным и без фиксации.
13. `REMOTE_OFF` должен иметь приоритет над LOCAL_START: при разомкнутой аварийной цепи удержание катушки невозможно.
14. Восстановление BMS или цепи REMOTE_OFF не должно автоматически включать контактор.
15. Конкретную модель контактора нельзя выбирать до формирования требований к катушке, economizer, вспомогательному контакту, силовому контакту и условиям эксплуатации.
16. Новые функции и дополнительная электроника не вводятся без отдельного решения.

## 9. Схемотехнический рубеж

Следующий блокирующий этап:

1. определить диапазон защищённого напряжения АКБ;
2. определить напряжение включения, удержания и отпускания катушки;
3. определить пусковой ток, ток удержания и тепловыделение;
4. определить требования к economizer;
5. определить рейтинг вспомогательного контакта;
6. определить минимальное время размыкания REMOTE_OFF;
7. определить защиту катушки без недопустимого увеличения времени отпускания;
8. определить силовой ток и DC breaking capacity;
9. определить отказобезопасное поведение и испытания;
10. только после этого выбирать конкретную модель контактора.
