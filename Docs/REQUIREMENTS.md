# Требования к ПДУ БНПА / PlataVM

Статус:

```text
requirements baseline for V1.5 battery architecture clarification
```

## 1. Верхнеуровневое требование

Плата должна безопасно распределять энергию от двух основных АКБ и резервного источника на 12 В выходы, 5 В выходы, световую ветвь и критическую управляющую подсистему.

## 2. Требования к основной батарейной подсистеме

| ID | Требование | Статус |
|---|---|---|
| REQ-BAT-001 | Поддержать АКБ_1/АКБ_2 LiFePO4 4s24p | принято |
| REQ-BAT-002 | Обе исправные АКБ в RUN должны работать параллельно | принято |
| REQ-BAT-003 | Не отключать исправную АКБ только по причине меньшего напряжения или SoC | принято |
| REQ-BAT-004 | Не использовать функцию BATTERY_MISMATCH | принято |
| REQ-BAT-005 | Реализовать отдельные симметричные Battery Front-End | принято |
| REQ-BAT-006 | Использовать однополюсные K_BAT1/K_BAT2 в положительных проводниках | принято |
| REQ-BAT-007 | Использовать СН-176А-12 как единственный рабочий межкорпусный вариант | принято |
| REQ-BAT-008 | Длина каждой батарейной кабельной сборки — 1 м | принято |
| REQ-BAT-009 | Реализовать измерения V/I/T и PACK_PRESENT каждой АКБ | принято |
| REQ-BAT-010 | Реализовать управляемый двунаправленный основной силовой тракт каждой ветви | принято |
| REQ-BAT-011 | Реализовать ограниченный балансировочный тракт каждой ветви | принято |

## 3. Требования к PACK_BUS и отключению

| ID | Требование | Статус |
|---|---|---|
| REQ-PWR-001 | Сформировать PACK_BUS как главную силовую шину после двух Battery Front-End | принято |
| REQ-PWR-002 | Не использовать отдельный K_MAIN | принято |
| REQ-PWR-003 | Не использовать отдельное имя MAIN_INPUT_BUS | принято |
| REQ-PWR-004 | Реализовать распределённый BATTERY_DISCONNECT через K_BAT1/K_BAT2 и электронные силовые тракты | принято |
| REQ-PWR-005 | Штатный HARD_OFF: снять тяжёлые нагрузки, сохранить журнал, отключить K_BAT1/K_BAT2 | принято |
| REQ-PWR-006 | Аварийный HARD_OFF: немедленно отключить обе батарейные ветви, затем записать причину от EMG | принято |
| REQ-PWR-007 | Контролировать BAT1_I, BAT2_I и разряд PACK_BUS после отключения | принято |

|---|---|---|
| REQ-P12-001 | Сформировать POWER_12V_BUS CH1...CH14 | принято |
| REQ-P12-002 | CH1...CH11 сделать MCU-controlled | принято |
| REQ-P12-003 | CH12...CH14 сделать Always-On monitored | принято |

| REQ-P12-006 | Always-On каналы должны отключаться защитой, SAFE и HARD_OFF | принято |

| REQ-5V-001 | Сформировать отдельную 5V_SYS_BUS от PACK_BUS через DC/DC | принято |

| REQ-5V-003 | Максимальный ток одного 5V_OUT — 3 А | принято |

## 8. Требования к интерфейсам и безопасности

| ID | Требование | Статус |
|---|---|---|
| REQ-IF-001 | Использовать isolated RS-485 как baseline | принято |

| REQ-SAFE-001 | EXT_KILL должен работать независимо от MCU | принято |

5. Пиковый ток и длительность пика CH1...CH14.
6. Точная топология LED-драйверов.
7. MCU.
8. Зарядный ток EMG.
9. Механика модулей и типы внутренних выходных разъёмов.
