# Управление проектом PlataVM

Документ задаёт правила ведения проекта ПДУ БНПА / PlataVM.

## 1. Назначение

PlataVM ведётся как аппаратный инженерный проект платы питания / энергетического модуля БНПА.

Каждое изменение должно отвечать на вопросы:

1. Что изменено?
2. Почему это изменено?
3. Какая часть аппаратной архитектуры затронута?
4. Не нарушены ли прежние зафиксированные решения?
5. Как проверено изменение?

## 2. Как восстановить состояние проекта после паузы

Любой будущий участник или ИИ-ассистент должен начинать работу строго в таком порядке:

1. Открыть `README.md`.
2. Открыть `Docs/PROJECT_MASTER.md`.
3. Открыть `Docs/ARCHITECTURE_BASELINE.md`.
4. Открыть `Docs/POWER_TREE_V1_5.md`.
5. Открыть `Docs/OPEN_QUESTIONS.md`.
6. Открыть `Docs/CHRONOLOGY.md`.
7. Проверить открытые PR и issues.

После этого будет понятно:

1. что проект — это ПДУ БНПА / PlataVM;
2. что текущая версия — V1.5 integrated TO-BE baseline;
3. что следующий рубеж — schematic freeze V1.5;
4. какие вопросы ещё мешают переходу к схеме;
5. какие решения нельзя менять без ADR.

## 3. Приоритет документов

1. `Docs/PROJECT_MASTER.md` — верхний уровень проекта.
2. `Docs/ARCHITECTURE_BASELINE.md` — базовая архитектура.
3. `Docs/POWER_TREE_V1_5.md` — дерево питания.
4. `Docs/REQUIREMENTS.md` — требования.
5. `Docs/INTERFACES_AND_SIGNALS.md` — интерфейсы и сигналы.
6. `Docs/FAULT_LOGIC_AND_STATES.md` — состояния и аварии.
7. `Docs/OPEN_QUESTIONS.md` — незакрытые вопросы.
8. `Docs/CHRONOLOGY.md` — история изменений.
9. `Docs/adr/` — архитектурные решения.

## 4. Архитектурные запреты без ADR

Без отдельного ADR нельзя:

1. питать POWER_12V_BUS от EMG;
2. питать 5V_SYS_BUS от EMG;
3. питать LIGHT_POWER_BRANCH от EMG;
4. убирать BATTERY_DISCONNECT;
5. убирать PRECHARGE;
6. заменять RS-485 как baseline;
7. возвращать TRACO как целевую архитектуру новой платы;
8. объединять 5V_SYS_BUS и 5V_CRIT;
9. переходить к PCB без schematic freeze.

## 5. Правило открытых вопросов

Вопросы к владельцу проекта ведутся в `Docs/OPEN_QUESTIONS.md`.

Если ответ меняет архитектуру, нужно обновить:

1. `Docs/ARCHITECTURE_BASELINE.md`;
2. `Docs/REQUIREMENTS.md`;
3. `Docs/CHRONOLOGY.md`;
4. ADR, если решение принципиальное.
