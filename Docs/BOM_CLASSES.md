# Классы компонентов и BOM-логика ПДУ БНПА / PlataVM

Дата фиксации: 2026-06-30

Статус:

```text
BOM classes baseline for V1.4 chat-consolidated architecture
```

## 1. Назначение

Документ фиксирует не финальный BOM, а классы компонентов, которые должны быть выбраны перед схемотехническим freeze и PCB.

Финальный BOM с артикулами формируется позже в `Manufacturing/BOM/`.

## 2. Общие правила выбора компонентов

Каждый компонент должен иметь:

1. назначение;
2. электрические параметры;
3. запас по напряжению;
4. запас по току;
5. запас по мощности/теплу;
6. температурный диапазон;
7. корпус;
8. доступность для закупки;
9. альтернативу;
10. связь с тестом.

Компонент нельзя считать выбранным, если неизвестны режимы работы, тепловая нагрузка и отказовая реакция.

## 4. BAT1/BAT2 компоненты

### 4.1. Разъёмы BAT

Требования:

1. выдерживать максимальный рабочий и пиковый ток;
2. иметь механическую фиксацию;
3. быть пригодными для эксплуатации в составе герметичного оборудования;
4. иметь понятную полярность;
5. исключать случайное перепутывание BAT1/BAT2, если это критично;
6. иметь запас по температуре.

### 4.2. Предохранители

Должны быть рассчитаны по:

1. максимальному штатному току;
2. пиковым токам при включении;
3. току короткого замыкания;
4. времени срабатывания;
5. селективности с BMS;
6. тепловому режиму.

### 4.3. MOSFET силовых ключей

Для основных ключей требуется оценить:

1. Vds;
2. Id continuous;
3. Id pulse;
4. Rds(on);
5. SOA;
6. корпус и тепловое сопротивление;
7. возможность back-to-back включения;
8. драйвер затвора;
9. поведение при отказе.

## 5. Precharge компоненты

Precharge-резистор выбирается по:

1. энергии заряда входной ёмкости BAT_BUS;
2. допустимому току;
3. времени предзаряда;
4. импульсной мощности;
5. температурному запасу;
6. отказовому поведению.

Precharge-ключ выбирается по:

1. напряжению;
2. току;
3. импульсной нагрузке;
4. управлению от MCU/драйвера;
5. защите от ошибочного состояния.

## 6. EMG компоненты

Для EMG нужны:

1. разъём EMG;
2. ключ EMG_MAIN;
3. ключ или контроллер заряда EMG;
4. ограничитель зарядного тока;
5. защита от обратного тока;
6. измерение напряжения EMG;
7. измерение температуры EMG;
8. BMS или интерфейс статуса EMG;
9. предохранитель или защита входа.

Открыто:

1. точный тип EMG-сборки;
2. допустимый зарядный ток;
3. BMS EMG;
4. режим обслуживания EMG.

## 7. Boost 12→32 В

Boost-преобразователь должен быть выбран по:

1. входному диапазону 4S LiFePO4;
2. выходу около 32 В;
3. мощности группы ИЗС;
4. пиковым токам;
5. КПД;
6. теплу;
7. защите от КЗ;
8. управлению EN;
9. диагностике power-good/fault;
10. доступности компонентов.

Для двух групп ИЗС возможны два независимых одинаковых boost-модуля или две одинаковые схемы на плате.

## 8. ИЗС-каналы

Для каждого ИЗС-канала нужны:

1. силовой ключ;
2. драйвер ключа или логический MOSFET;
3. токовая диагностика;
4. защита от КЗ;
5. защита от перегрева;
6. PWM-управление;
7. выходной разъём;
8. маркировка канала;
9. тестовая точка.

## 9. AUX-каналы

Финальный выбор AUX-компонентов невозможен без уточнения нагрузок.

До уточнения допускается только резервирование архитектуры:

1. ключ на канал;
2. ограничение тока;
3. защита от КЗ;
4. телеметрия состояния;
5. возможность отключения каналов независимо.

## 10. MCU

MCU выбирается по:

1. числу ADC-каналов;
2. числу PWM-каналов;
3. числу GPIO;
4. поддержке UART/RS-485 через UART;
5. наличию watchdog;
6. температурному диапазону;
7. доступности отладки SWD;
8. надёжности питания 3,3 В;
9. возможности логирования состояний;
10. запасу по памяти для fault manager.

## 11. RS-485

Трансивер RS-485 выбирается по:

1. напряжению питания;
2. ESD-стойкости;
3. допустимой скорости;
4. fail-safe функциям;
5. температурному диапазону;
6. защите от bus fault;
7. необходимости изоляции;
8. корпусу;
9. доступности.

## 12. Датчики тока

Возможные классы:

1. шунт + усилитель;
2. Hall sensor;
3. high-side current monitor;
4. smart power switch с диагностикой.

Выбор зависит от:

1. тока канала;
2. потерь;
3. точности;
4. общей земли;
5. цены;
6. наличия гальванической развязки;
7. требований к диагностике.

## 13. Датчики температуры

Температуру нужно контролировать минимум в зонах:

1. входные силовые ключи;
2. boost A;
3. boost B;
4. зона MCU/critical;
5. корпус/окружающая среда, если требуется.

## 14. Статусы BOM

Для каждого компонента в финальном BOM должен быть статус:

```text
candidate / selected / tested / approved / replaced / obsolete
```

## 15. Минимальный состав финального BOM

Финальный BOM должен содержать:

1. позиционное обозначение;
2. наименование;
3. производитель;
4. MPN;
5. поставщик;
6. артикул поставщика;
7. корпус;
8. номинал;
9. напряжение;
10. ток;
11. мощность;
12. температурный диапазон;
13. количество;
14. альтернативы;
15. статус проверки;
16. комментарий по риску.

## 16. Критерий готовности BOM к прототипу

BOM готов к прототипу, когда:

1. выбраны все силовые компоненты;
2. выбраны все critical-компоненты;
3. выбран MCU;
4. выбран RS-485;
5. выбран boost 32 В;
6. выбраны компоненты ИЗС;
7. выбраны защиты;
8. закрыты открытые вопросы, влияющие на номиналы;
9. для критических компонентов есть альтернативы;
10. BOM связан с тест-планом.
