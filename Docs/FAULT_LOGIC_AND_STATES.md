# Состояния, аварийная логика и реакции ПДУ БНПА / PlataVM

Статус:

```text
state-machine baseline for V1.5 monostable battery contactors
```

## 1. Состояния

| Состояние | Назначение |
|---|---|
| OFF | основные батарейные ветви отключены, K_BAT1/K_BAT2 OPEN |
| BOOT | запуск critical domain и MCU |
| SELF_TEST | самопроверка |
| DECK_BALANCE | палубное межблочное выравнивание АКБ_1/АКБ_2 и обслуживание EMG |
| RUN | штатная параллельная работа обеих АКБ |
| SAFE | ограничение нагрузки перед штатным отключением |
| KEEP_ALIVE | питание только критического домена от EMG |
| HARD_OFF | физическое отключение K_BAT1 и K_BAT2 разрывом hold loop |
| FAULT_LATCH | аварийная защёлка, повторный запуск запрещён до сброса |

Отдельное состояние для различия уровней заряда батарей не используется. Различия напряжений, токов и расчётного SoC являются диагностическими параметрами.

## 2. RUN

В `RUN` обе исправные АКБ работают параллельно через симметричные Battery Front-End. Различия напряжений, токов и расчётного SoC используются для телеметрии и расчёта состояния, но сами по себе не вызывают отключения батареи.

Условия RUN для каждой активной ветви:

1. BMS выдаёт защищённое напряжение;
2. K_BATx удерживается через K_BAT_AUX_NO;
3. REMOTE_OFF/EXT_KILL loop замкнут;
4. MAIN_SWx разрешён;
5. ток и напряжение находятся в допустимом диапазоне.

## 3. DECK_BALANCE

В режиме `DECK_BALANCE`:

1. LIGHT_POWER_BRANCH отключена;
2. POWER_12V_BUS отключена либо переведена в сервисный минимум;
3. 5V_SYS_BUS отключена либо ограничена сервисными нагрузками;
4. MCU контролирует BAT1_V, BAT2_V, BAT1_I, BAT2_I, внешние температуры и расчётный SoC;
5. разрешается ограниченный межблочный балансировочный ток;
6. операция прекращается при превышении тока, температуры или времени;
7. при необходимости выполняется заряд и диагностика EMG.

## 4. Штатный HARD_OFF

```text
SAFE
→ отключить тяжёлые нагрузки
→ сохранить журнал и SoC
→ разомкнуть BAT1_HOLD_LOOP
→ подтвердить K_BAT1 OPEN
→ разомкнуть BAT2_HOLD_LOOP
→ подтвердить K_BAT2 OPEN
→ проверить BAT1_I = 0 и BAT2_I = 0
→ проверить разряд PACK_BUS
→ завершить работу MCU от EMG
```

После отпускания контактора программный REMOTE_OFF может вернуться в неактивное состояние. Повторного включения нет, потому что K_BAT_AUX_NO разомкнут.

## 5. Аварийный HARD_OFF

При `EXT_KILL` или критической аварии:

1. аппаратно размыкаются hold loops K_BAT1/K_BAT2;
2. электронные силовые тракты MAIN_SW1/MAIN_SW2 отключаются немедленно;
3. EXT_KILL удерживает запрет запуска до ручного сброса;
4. причина события сохраняется после отключения основных АКБ за счёт EMG;
5. контролируются токи ветвей и разряд PACK_BUS.

## 6. Реакция на отключение BMS

Для каждой батарейной ветви:

```text
BMS protected output lost
→ K_BAT coil de-energized
→ K_BAT OPEN
→ K_BAT_AUX_NO OPEN
→ branch unavailable
```

После восстановления BMS:

```text
K_BAT remains OPEN
→ автоматический переход ветви в RUN запрещён
→ требуется новый LOCAL_START
→ затем SELF_TEST батарейной ветви
→ только после проверки разрешается MAIN_SWx
```

Если защищённый выход BMS восстанавливается во время `FAULT_LATCH` или активного `EXT_KILL`, LOCAL_START должен оставаться заблокирован разомкнутым hold loop.

## 7. Диагностика состояния K_BATx

Состояние оценивается по:

1. команде разрыва hold loop;
2. ожидаемому времени отпускания;
3. BATx_RAW_V;
4. BATx_I;
5. PACK_BUS_V;
6. реакции ветви на MAIN_SWx.

Контакт 12 не используется для feedback.

Основные диагностические признаки:

| Признак | Возможная причина |
|---|---|
| hold loop разомкнут, ток ветви сохраняется | сваривание главного контакта или обходной токовый путь |
| LOCAL_START выполнен, BATx_RAW_V не появилось | отказ катушки, контактора, BMS или цепи запуска |
| ток катушки повышен | межвитковое замыкание, отказ economizer, пониженное сопротивление |
| ток катушки отсутствует при LOCAL_START | обрыв F_CTRL, катушки, LOCAL_START, pin 11 или hold loop |
| контактор многократно переключается | напряжение около порога удержания, плохой контакт, недостаточная энергия катушки |
| после восстановления BMS ветвь включилась без LOCAL_START | залипший LOCAL_START, сваренный K_BAT_AUX_NO или ошибка проводки |

## 8. Запрещено

1. Питать POWER_12V_BUS от EMG.
2. Питать 5V_SYS_BUS от EMG.
3. Питать LIGHT_POWER_BRANCH от EMG.
4. Игнорировать EXT_KILL.
5. Выполнять межблочное выравнивание без ограничения тока.
6. Выполнять штатное выключение без проверки токов батарейных ветвей и PACK_BUS.
7. Автоматически запускать K_BATx после восстановления BMS.
8. Использовать raw-ветвь или внутренние точки BMS для катушки.
9. Разрешать LOCAL_START при активном EXT_KILL.
10. Считать снятие питания катушки достаточным доказательством размыкания при возможном сваривании главного контакта.

## 9. Аварийные реакции

| Авария | Реакция |
|---|---|
| EXT_KILL | разомкнуть оба hold loop, отключить MAIN_SW1/MAIN_SW2, перейти в FAULT_LATCH |
| Потеря защищённого выхода BMS | соответствующий K_BATx автоматически OPEN, отключить MAIN_SWx, записать событие |
| Неожиданное восстановление BMS | сохранить K_BATx OPEN, ожидать ручной LOCAL_START |
| Overcurrent BATx | отключить MAIN_SWx, разомкнуть соответствующий hold loop, зафиксировать отказ |
| K_BAT не отпустил после REMOTE_OFF | отключить MAIN_SWx, зафиксировать возможное сваривание контакта |
| K_BAT не включился после LOCAL_START | не разрешать MAIN_SWx, зафиксировать отказ запуска |
| Повторный запуск без LOCAL_START | немедленно отключить MAIN_SWx и hold loop, перейти в FAULT_LATCH |
| Coil overcurrent / economizer fault | разомкнуть hold loop, зафиксировать отказ |
| Hold-loop open circuit | K_BATx отпускает либо не запускается, записать отказ линии |
| Hold-loop short / OFF element failed | MAIN_SWx обеспечивает резервное отключение, зафиксировать отказ REMOTE_OFF |
| Balance overcurrent | прекратить DECK_BALANCE |
| Balance overtemperature | прекратить DECK_BALANCE |
| Balance timeout | прекратить DECK_BALANCE, сохранить событие |
| Overcurrent CHx | отключить канал, записать fault |
| 5V_SYS fault | отключить 5V_SYS, сохранить critical domain |
| LED fault | отключить аварийный LED-канал |
| RS-485 timeout | перейти в безопасную политику |
| EMG undervoltage | ограничить keep-alive, записать fault |
