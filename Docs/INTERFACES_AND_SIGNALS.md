# Интерфейсы, шины и сигналы ПДУ БНПА / PlataVM

Статус:

```text
interfaces baseline for V1.5 monostable battery contactors
```

## 1. Основной внешний интерфейс

```text
isolated RS-485 half-duplex
```

RS-485 используется между ПДУ и верхним уровнем, но не проходит в корпуса основных АКБ.

CAN/CAN-FD не входит в baseline V1.5 и допускается только как optional/future.

## 2. Энергетические шины

| Шина | Источник | Назначение |
|---|---|---|
| PACK_BUS | BFE_1/BFE_2 | главная силовая шина после двух АКБ |
| POWER_12V_BUS | PACK_BUS | CH1...CH14 |
| 5V_SYS_BUS | PACK_BUS через DC/DC | 5V_OUT1...5V_OUT10 |
| LIGHT_POWER_BRANCH | PACK_BUS | LED-драйверы |
| RESERVE_BRANCH | PACK_BUS + EMG | заряд EMG и keep-alive |
| 5V_CRIT | critical power path | MCU/связь/датчики |
| 3V3_CRIT | 5V_CRIT | цифровая логика |

## 3. Межкорпусная линия каждой АКБ

Через СН-176А-12 передаются только:

1. параллельная группа `BAT+` на контактах 1–5;
2. параллельная группа `BAT−` на контактах 6–10;
3. `REMOTE_OFF / K_BAT_HOLD_RETURN` на контакте 11.

Контакт 12 обозначается `RESERVE` и не назначен.

Не передаются `LOCAL_START`, RS-485, CAN, UART, цифровая телеметрия BMS, локальная MCU-связь, raw-питание, внутренние сигналы BMS или вторая линия катушки.

## 4. Граница BMS

BMS является закрытым автономным узлом. Для системы доступны только защищённые внешние полюса:

```text
BAT_PROT+
BAT_PROT−
```

Сигнальная архитектура не зависит от high-side или low-side исполнения BMS.

Предварительная электрическая рамка управляющей цепи:

```text
0...16,0 VDC continuous withstand
pickup: 9,0...16,0 V
guaranted hold after pull-in: 7,5...16,0 V
```

## 5. Локальная линия каждой АКБ

На корпусе каждой АКБ установлен отдельный двухконтактный разъём `LOCAL_START`.

Он используется для кратковременного ручного замыкания цепи запуска, включённой параллельно `K_BAT_AUX_NO`.

`LOCAL_START`:

1. нормально разомкнут;
2. пружинно-возвратный;
3. без фиксации;
4. не является интерфейсом связи;
5. не может преодолеть разомкнутую цепь `REMOTE_OFF / EXT_KILL`;
6. предварительно рассчитан не менее чем на 0,5 А при 16 В DC и 2 А / 150 мс.

## 6. Электрические пределы pin 11

Контакт 11 является токовой частью цепи катушки:

| Параметр | Требование |
|---|---:|
| Continuous current | ≤0,25 А |
| Pulsed current | ≤2,0 А / 150 мс |
| Normal voltage | 0…16 В DC |
| Preliminary clamp transient | 24…36 В |
| Closed path voltage drop | ≤0,2 В при hold current |
| Open-state leakage | ≤1 мА |

Конкретная кабельная сборка должна подтвердить эти значения испытанием.

## 7. Сигналы батарейных ветвей в корпусе электроники

Для АКБ_1:

1. `BAT1_REMOTE_OFF_REQ` — программный запрос разрыва цепи удержания;
2. `BAT1_HOLD_LOOP_OPEN_CMD` — управление дистанционным размыкающим элементом;
3. `BAT1_MAIN_SW_EN`;
4. `BAT1_BALANCE_SW_EN`;
5. `BAT1_PRESENT_SENSE`;
6. `BAT1_V`;
7. `BAT1_I`;
8. `BAT1_RAW_V` после межкорпусного соединителя.

Для АКБ_2 используются симметричные сигналы `BAT2_*`.

Физический путь `REMOTE_OFF` проходит только по контакту 11 соответствующего СН-176А-12. Дополнительные командные или обратные линии к корпусу АКБ не вводятся.

## 8. Семантика REMOTE_OFF

`REMOTE_OFF` является командой разрыва цепи единственной катушки контактора.

```text
BATx_REMOTE_OFF_REQ active
→ BATx_HOLD_LOOP_OPEN_CMD active
→ цепь контакта 11 разомкнута
→ катушка обесточена
→ K_BATx OPEN
```

Штатная команда должна удовлетворять:

```text
t_OFF ≥ max(250 ms; 5 × t_release_max)
```

Предварительная цель:

```text
t_release_system ≤ 100 ms
```

После подтверждённого отпускания программный размыкающий элемент может вернуться в исходное состояние. Контактор не включается повторно, потому что `K_BAT_AUX_NO` разомкнут.

Для аварийного запрета `EXT_KILL` удерживает цепь разомкнутой до ручного сброса.

## 9. Требования к размыкающему элементу REMOTE_OFF

| Параметр | Предварительное требование |
|---|---:|
| DC contact rating | ≥30 В DC, ≥0,5 А continuous, ≥2 А pulsed |
| Semiconductor blocking rating | ≥60 В |
| Closed-state drop | ≤0,2 В при hold current |
| Open-state leakage | ≤1 мА |
| Fail behavior | потеря питания управляющей электроники не оставляет опасное CLOSED |

Точная реализация — реле, полупроводниковый узел или иной DC-коммутатор — остаётся схемотехническим вопросом.

## 10. Контроль результата переключения

Базовый контроль выключения выполняется по:

1. факту разрыва цепи удержания;
2. выдержке `t_OFF`;
3. ожидаемому и измеренному времени отпускания;
4. `BATx_RAW_V`;
5. `BATx_I`;
6. влиянию ветви на `PACK_BUS_V`.

Контакт 12 не используется для auxiliary feedback.

## 11. Сигналы HARD_OFF

1. `EXT_KILL` — независимый аппаратный запрос аварийного отключения.
2. `HARD_OFF_REQ` — программный запрос штатного завершения.
3. `BAT1_REMOTE_OFF_REQ`.
4. `BAT2_REMOTE_OFF_REQ`.
5. `BAT1_HOLD_LOOP_OPEN_CMD`.
6. `BAT2_HOLD_LOOP_OPEN_CMD`.
7. `BAT1_MAIN_SW_EN = 0`.
8. `BAT2_MAIN_SW_EN = 0`.
9. `PACK_BUS_DISCHARGE_EN`, если используется управляемый разряд.
10. `PACK_BUS_V` — контроль остаточного напряжения.

`EXT_KILL` аппаратно размыкает обе цепи удержания и отключает электронные силовые тракты независимо от MCU.

## 12. Поведение при отключении BMS

```text
BMS output disabled
→ coil de-energized
→ K_BATx OPEN

BMS output restored
→ K_BAT_AUX_NO OPEN
→ no automatic restart
```

Цифровой сигнал от BMS не требуется.

## 13. Сигналы POWER_12V_BUS

1. `CH1_EN` ... `CH11_EN`.
2. `CH1_FAULT` ... `CH14_FAULT`.
3. `CH1_ISENSE` ... `CH14_ISENSE`.
4. `CH12...CH14` — Always-On monitored.

## 14. Сигналы 5V_SYS_BUS

1. `5V_SYS_EN`.
2. `5V_OUT1_EN` ... `5V_OUT7_EN`.
3. `5V_OUT1_FAULT` ... `5V_OUT10_FAULT`.
4. `5V_OUT1_ISENSE` ... `5V_OUT10_ISENSE`.
5. `5V_OUT8...5V_OUT10` — Always-On monitored.

## 15. Сигналы LIGHT_POWER_BRANCH

1. `LED1_PWM` ... `LED6_PWM`.
2. `LED1_FAULT` ... `LED6_FAULT`.
3. `LED1_ISENSE` ... `LED6_ISENSE`.
4. `LIGHT_BRANCH_EN`.

## 16. Минимальная телеметрия

1. BAT1_V/I/PRESENT и факт последней команды REMOTE_OFF.
2. BAT2_V/I/PRESENT и факт последней команды REMOTE_OFF.
3. измеренное время отключения каждой ветви.
4. PACK_BUS_V и суммарный ток.
5. состояния CH1...CH14 и токи каналов.
6. состояния 5V_OUT1...5V_OUT10 и токи каналов.
7. состояния LED1...LED6 и токи каналов.
8. состояние EMG.
9. режим FSM.
10. fault-коды и журнал событий.
11. состояние внешнего RS-485.

Внутренняя температура ячеек и цифровой статус BMS не входят в доступную телеметрию текущей архитектуры.

## 17. Открытые электрические параметры

1. фактические пороги BAT_PROT и характер отключения/восстановления BMS;
2. dropout и brownout-поведение кандидата;
3. индуктивность и энергия катушки;
4. реализация `REMOTE_OFF_NC / EXT_KILL_NC`;
5. конкретный suppression;
6. окончательный F_CTRL;
7. тепловой режим;
8. реакция на короткое замыкание pin 11, сваривание и дребезг;
9. стендовые результаты кандидатов.

Полная расчётная рамка приведена в `Docs/KBAT_ELECTRICAL_REQUIREMENTS.md`.
