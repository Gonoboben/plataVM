# Интерфейсы, шины и сигналы ПДУ БНПА / PlataVM

Статус:

```text
interfaces baseline for V1.5 bistable battery contactors
```

## 1. Основной внешний интерфейс

```text
isolated RS-485 half-duplex
```

RS-485 используется между ПДУ и верхним уровнем, но не проходит в корпуса основных АКБ.

CAN/CAN-FD не входит в baseline V1.5 и допускается только как optional/future.

## 2. Энергетические шины

| Шина | Источник | Назначение |
|---|---|---|
| PACK_BUS | BFE_1/BFE_2 | главная силовая шина после двух АКБ |
| POWER_12V_BUS | PACK_BUS | CH1...CH14 |
| 5V_SYS_BUS | PACK_BUS через DC/DC | 5V_OUT1...5V_OUT10 |
| LIGHT_POWER_BRANCH | PACK_BUS | LED-драйверы |
| RESERVE_BRANCH | PACK_BUS + EMG | заряд EMG и keep-alive |
| 5V_CRIT | critical power path | MCU/связь/датчики |
| 3V3_CRIT | 5V_CRIT | цифровая логика |

## 3. Межкорпусная линия каждой АКБ

Через СН-176А-12 передаются только:

1. параллельная группа `BAT+` на контактах 1–5;
2. параллельная группа `BAT−` на контактах 6–10;
3. кратковременный импульс `REMOTE_OFF` на контакте 11.

Контакт 12 обозначается `RESERVE` и не назначен.

Не передаются:

1. `LOCAL_START`;
2. RS-485;
3. CAN;
4. UART;
5. цифровая телеметрия BMS;
6. локальная MCU-связь;
7. постоянное питание катушки.

## 4. Локальная линия каждой АКБ

На корпусе каждой АКБ установлен отдельный двухконтактный разъём `LOCAL_START`.

Он используется только для кратковременного ручного замыкания цепи включающего импульса контактора.

`LOCAL_START` не является интерфейсом связи и не должен оставаться постоянно замкнутым.

## 5. Сигналы батарейных ветвей в корпусе электроники

Для АКБ_1:

1. `BAT1_REMOTE_OFF_REQ` — запрос на формирование конечного импульса;
2. `BAT1_MAIN_SW_EN`;
3. `BAT1_BALANCE_SW_EN`;
4. `BAT1_PRESENT_SENSE`;
5. `BAT1_V`;
6. `BAT1_I`;
7. `BAT1_RAW_V` после межкорпусного соединителя.

Для АКБ_2 используются симметричные сигналы `BAT2_*`.

Физический импульс `REMOTE_OFF` передаётся только по контакту 11 соответствующего СН-176А-12. Дополнительные командные или обратные линии к корпусу АКБ не вводятся.

## 6. Контроль результата переключения

Базовый контроль выключения выполняется по:

1. факту запуска конечного импульса `REMOTE_OFF`;
2. исчезновению или изменению `BATx_RAW_V`;
3. току `BATx_I`;
4. влиянию ветви на `PACK_BUS_V`.

Контакт 12 не используется для auxiliary feedback в текущей архитектуре.

## 7. Сигналы HARD_OFF

1. `EXT_KILL` — независимый аппаратный запрос аварийного отключения.
2. `HARD_OFF_REQ` — программный запрос штатного завершения.
3. `BAT1_REMOTE_OFF_REQ`.
4. `BAT2_REMOTE_OFF_REQ`.
5. `BAT1_MAIN_SW_EN = 0`.
6. `BAT2_MAIN_SW_EN = 0`.
7. `PACK_BUS_DISCHARGE_EN`, если используется управляемый разряд.
8. `PACK_BUS_V` — контроль остаточного напряжения.

`EXT_KILL` должен аппаратно отключать электронные силовые тракты и инициировать оба конечных импульса `REMOTE_OFF` независимо от MCU.

## 8. Сигналы POWER_12V_BUS

1. `CH1_EN` ... `CH11_EN`.
2. `CH1_FAULT` ... `CH14_FAULT`.
3. `CH1_ISENSE` ... `CH14_ISENSE`.
4. `CH12...CH14` — Always-On monitored.

## 9. Сигналы 5V_SYS_BUS

1. `5V_SYS_EN`.
2. `5V_OUT1_EN` ... `5V_OUT7_EN`.
3. `5V_OUT1_FAULT` ... `5V_OUT10_FAULT`.
4. `5V_OUT1_ISENSE` ... `5V_OUT10_ISENSE`.
5. `5V_OUT8...5V_OUT10` — Always-On monitored.

## 10. Сигналы LIGHT_POWER_BRANCH

1. `LED1_PWM` ... `LED6_PWM`.
2. `LED1_FAULT` ... `LED6_FAULT`.
3. `LED1_ISENSE` ... `LED6_ISENSE`.
4. `LIGHT_BRANCH_EN`.

## 11. Минимальная телеметрия

1. BAT1_V/I/PRESENT и факт последней команды REMOTE_OFF.
2. BAT2_V/I/PRESENT и факт последней команды REMOTE_OFF.
3. PACK_BUS_V и суммарный ток.
4. состояния CH1...CH14 и токи каналов.
5. состояния 5V_OUT1...5V_OUT10 и токи каналов.
6. состояния LED1...LED6 и токи каналов.
7. состояние EMG.
8. режим FSM.
9. fault-коды и журнал событий.
10. состояние внешнего RS-485.

Внутренняя температура ячеек и цифровой статус BMS не входят в доступную телеметрию текущей архитектуры.

## 12. Неопределённые электрические параметры

До схемотехнической фиксации должны быть определены:

1. электрическая схема импульса `LOCAL_START`;
2. электрическая схема импульса `REMOTE_OFF`;
3. длительность импульса;
4. полярность импульса;
5. ток и напряжение катушки;
6. реакция на удержание команды и повторные импульсы.
