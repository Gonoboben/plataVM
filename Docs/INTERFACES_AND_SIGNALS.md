# Интерфейсы, шины и сигналы ПДУ БНПА / PlataVM

Дата фиксации: 2026-06-30

Статус:

```text
interfaces baseline for V1.4 chat-consolidated architecture
```

## 1. Назначение

Документ фиксирует рабочую карту шин, интерфейсов и управляющих сигналов ПДУ БНПА.

Он должен использоваться при:

1. проектировании электрической схемы;
2. выборе MCU и периферии;
3. разработке прошивки;
4. разводке PCB;
5. подготовке тестов.

## 6. Внешние интерфейсы

### 6.1. RS-485

Основной внешний интерфейс:

```text
RS-485
```

Назначение:

1. передача телеметрии;
2. приём команд включения/отключения каналов;
3. передача fault-кодов;
4. сервисная диагностика в составе аппарата.

Для RS-485 должны быть определены:

1. скорость обмена;
2. протокол;
3. адрес устройства;
4. формат телеметрии;
5. формат команд;
6. терминаторы;
7. fail-safe biasing;
8. ESD/Surge защита;
9. гальваническая развязка — по решению после оценки схемы подключения.

### 6.2. UART

UART — сервисный интерфейс.

Назначение:

1. первичная отладка;
2. логирование на стенде;
3. диагностика прошивки;
4. загрузка сервисных команд при отладке.

UART не является основным рабочим каналом аппарата.

### 6.3. SWD

SWD — интерфейс программирования и отладки MCU.

Требования:

1. доступен на сервисном разъёме;
2. защищён от случайного подключения к силовым линиям;
3. расположен вне высокотоковой зоны;
4. имеет понятную маркировку на PCB.

### 6.4. CAN

CAN пока рассматривается как опциональный интерфейс.

Ввод CAN требует отдельного архитектурного решения и обновления:

1. требований;
2. схемы;
3. протокола;
4. PCB-правил;
5. тест-плана.

## 7. Минимальный набор телеметрии

ПДУ должен передавать:

1. напряжение BAT1;
2. напряжение BAT2;
3. ток BAT1;
4. ток BAT2;
5. напряжение BAT_BUS_4S;
6. напряжение EMG;
7. состояние заряда/готовности EMG;
8. состояние 5V_CRIT;
9. состояние 3V3_CRIT;
10. состояние BOOST_A;
11. состояние BOOST_B;
12. состояния ИЗС 1..6;
13. состояния AUX1..AUX4;
14. температуры силовых зон;
15. активный режим конечного автомата;
16. активные fault-коды;
17. статус fault latch.

## 8. Минимальный набор команд

ПДУ должен принимать команды:

1. перейти в SELF_TEST;
2. включить основную шину;
3. отключить основную шину;
4. включить/выключить BOOST_A;
5. включить/выключить BOOST_B;
6. задать яркость ИЗС 1..6;
7. включить/выключить AUX1..AUX4;
8. запросить телеметрию;
9. сбросить разрешённые fault-коды;
10. перейти в безопасное состояние;
11. разрешить/запретить заряд EMG.

Команда сброса аварии не должна сбрасывать аппаратный fault latch без выполнения условий безопасности.

## 9. Проверки интерфейсов

До релиза должны быть проверены:

1. обмен RS-485 без нагрузки;
2. обмен RS-485 при работе ИЗС;
3. обмен RS-485 при переключении boost;
4. обмен RS-485 при аварии BAT1/BAT2;
5. обмен RS-485 при переходе на EMG;
6. устойчивость UART/SWD к шумам в стендовой среде;
7. корректность телеметрии по всем измеряемым каналам;
8. отсутствие ложных команд при сбросе MCU.
