pdu-master-concept-v1-4-2026-06-30

Что изменено:

1. Обновлён `README.md` как входная точка проекта.
2. Добавлен `Docs/PROJECT_MASTER.md`.
3. Обновлён `Docs/ARCHITECTURE_BASELINE.md` до статуса `V1.4 chat-consolidated baseline`.
4. Добавлен `Docs/REQUIREMENTS.md`.
5. Добавлен `Docs/INTERFACES_AND_SIGNALS.md`.

11. Добавлен `Docs/OPEN_QUESTIONS.md`.
12. Добавлен ADR по фиксации мастер-концепции.

Зачем изменено:

Собрать проект платы БНПА из материалов чата в единую рабочую структуру, пригодную для дальнейшей разработки через GitHub, без повторного начала проекта и без потери уже принятых решений.

Затронутая архитектура PlataVM:

Архитектура не пересобиралась заново. Зафиксирована рабочая консолидация:

1. BAT1/BAT2 — LiFePO4 4S, 12,8 В, по 144 А·ч;
2. основная шина — BAT_BUS_4S;
3. суммарная основная ёмкость — 288 А·ч;
4. EMG_4S2P — отдельный критический резерв, рабочее уточнение 12 А·ч;
5. EMG не питает ИЗС, BOOST 32 В и силовые AUX;
6. ИЗС — 6 каналов по 30 Вт;
7. диммирование ИЗС — от 5%;
8. базовый интерфейс — RS-485;
9. сервисные интерфейсы — UART/SWD;
10. следующий инженерный рубеж — schematic freeze V1.4.

Проверка:

1. Изменения касаются только документации.
2. Электрическая схема не добавлялась.
3. PCB не добавлялась.
4. BOM с артикулами не добавлялся.
5. Прошивка не добавлялась.
6. Открытые вопросы вынесены в отдельный файл.
7. Документы связаны между собой через `PROJECT_MASTER.md`.

Что специально не изменялось:

1. Схема питания физически не менялась.
2. Компоненты не выбирались.
3. PCB не разводилась.
4. Номиналы силовых элементов не утверждались.
5. Зарядный ток EMG не утверждался.
6. Тип MCU не утверждался.
7. Габариты платы не утверждались.

Следующий шаг:

Закрыть вопросы из `Docs/OPEN_QUESTIONS.md`, влияющие на схемотехнический freeze, затем перейти к `schematic freeze V1.4`.
