# ADR-2026-07-07: Распределённое отключение АКБ и режим DECK_BALANCE

## 1. Контекст

В V1.5 первоначально сохранялись центральный `K_MAIN`, отдельная шина `MAIN_INPUT_BUS` и логика предварительной блокировки батарейной ветви при различии напряжений.

После уточнения реальной эксплуатационной логики принято:

1. обе основные АКБ работают параллельно в миссии;
2. после миссии аппарат находится на палубе;
3. выравнивание состояния АКБ_1 и АКБ_2 выполняется в отдельном режиме MCU `DECK_BALANCE`;
4. отдельная функция `BATTERY_MISMATCH` не требуется;
5. каждая АКБ имеет собственный контактор в корпусе и собственный Battery Front-End.

## 2. Принятое решение

1. Удалить `BATTERY_MISMATCH` как состояние, fault и причину отключения исправной АКБ.
2. Оставить измерение BAT1_V/BAT2_V, BAT1_I/BAT2_I, температур и SoC для телеметрии, защиты и DECK_BALANCE.
3. Реализовать обе батарейные ветви симметрично.
4. Использовать однополюсные контакторы `K_BAT1` и `K_BAT2` в положительных проводниках.
5. Использовать `СН-176А-12` как единственный рабочий межкорпусный вариант каждой основной АКБ.
6. Использовать кабельную сборку длиной 1 м для каждой батарейной линии.
7. Удалить центральный `K_MAIN`.
8. Удалить отдельное имя `MAIN_INPUT_BUS`.
9. Использовать `PACK_BUS` как единственную главную силовую шину после двух Battery Front-End.
10. Реализовать `BATTERY_DISCONNECT` распределённо через K_BAT1/K_BAT2 и электронные силовые тракты.
11. Штатный HARD_OFF выполнять после сохранения журнала командами OFF на K_BAT1/K_BAT2.
12. Аварийный HARD_OFF выполнять немедленным отключением обеих батарейных ветвей с последующей записью причины от EMG.
13. Реализовать DECK_BALANCE через ограниченные балансировочные тракты с контролем тока, температуры и времени.

## 3. Каноническая структура

```text
АКБ_1 → BMS → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                      ├→ PACK_BUS
АКБ_2 → BMS → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘

PACK_BUS
├→ POWER_12V_BUS
├→ 5V_SYS_BUS
├→ LIGHT_POWER_BRANCH
└→ RESERVE_BRANCH
```

## 4. Последствия

Положительные последствия:

1. меньше силовых элементов в общем тракте;
2. меньше падение напряжения и нагрев;
3. K_BAT1/K_BAT2 обеспечивают физическое отключение источников;
4. электронные ключи обеспечивают быструю защиту и мягкие режимы;
5. DECK_BALANCE соответствует ранее принятой эксплуатационной логике;
6. архитектура остаётся симметричной для двух АКБ.

Риски и ограничения:

1. HARD_OFF зависит от гарантированной команды OFF обоим контакторам;
2. необходима обратная связь положения контакторов;
3. необходим гарантированный запас энергии для отключающего импульса;
4. после отключения требуется контроль разряда PACK_BUS;
5. допустимый ток СН-176А-12 и распайка контактов ещё не подтверждены;
6. балансировочный тракт требует отдельного расчёта по BMS и теплу.

## 5. Что не определено этим ADR

1. конкретная модель K_BAT1/K_BAT2;
2. тип электронного силового ключа Battery Front-End;
3. распайка СН-176А-12;
4. допустимый ток батарейной линии;
5. номинал и мощность балансировочного резистора;
6. критерии завершения DECK_BALANCE.

## 6. Следующий шаг

Определить распайку СН-176А-12 и расчётный ток одной батарейной линии, затем выполнить расчёт балансировочного тракта.
