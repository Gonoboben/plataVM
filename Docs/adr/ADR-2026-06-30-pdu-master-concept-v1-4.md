# ADR-2026-06-30: Фиксация мастер-концепции ПДУ БНПА V1.4

Дата: 2026-06-30

Статус:

```text
accepted
```

## 1. Контекст

Проект платы питания БНПА развивался в чате через несколько архитектурных уточнений.

К моменту фиксации сформирована рабочая архитектура, основанная на файле ПДУ БНПА V1.3 и последующих уточнениях из чата проекта.

Главная проблема перед этим решением:

```text
контекст проекта был распределён между сообщениями, вложенным файлом и частичными документами
```

Из-за этого существовал риск:

1. повторно начинать проект с нуля;
2. потерять уже принятые решения;
3. смешать архитектурную базу с открытыми вопросами;
4. перейти к PCB без достаточной фиксации требований;
5. сделать схему, которая не соответствует аварийной логике.

## 2. Решение

Принято зафиксировать `V1.4 chat-consolidated baseline` как рабочую архитектурную базу проекта.

Для этого создан управляющий пакет документов:

1. `Docs/PROJECT_MASTER.md`;
2. `Docs/ARCHITECTURE_BASELINE.md`;
3. `Docs/REQUIREMENTS.md`;
4. `Docs/INTERFACES_AND_SIGNALS.md`;
5. `Docs/FAULT_LOGIC_AND_STATES.md`;
6. `Docs/PCB_RULES.md`;
7. `Docs/BOM_CLASSES.md`;
8. `Docs/TEST_PLAN.md`;
9. `Docs/IMPLEMENTATION_ROADMAP.md`;
10. `Docs/OPEN_QUESTIONS.md`;
11. `Docs/CHRONOLOGY.md`.

## 3. Зафиксированные архитектурные решения

1. BAT1 и BAT2 — основные аккумуляторные входы LiFePO4 4S.
2. Номинальное напряжение основной системы — 12,8 В.
3. BAT1 — 144 А·ч.
4. BAT2 — 144 А·ч.
5. Суммарная основная ёмкость — 288 А·ч.
6. BAT1 и BAT2 работают через управляемое параллельное подключение.
7. Перед параллельным подключением требуется контроль напряжений и precharge.
8. Основная силовая шина — BAT_BUS_4S.
9. EMG_4S2P является отдельным критическим резервом.
10. Рабочее уточнение ёмкости EMG — 12 А·ч.
11. EMG не питает ИЗС.
12. EMG не питает BOOST 32 В.
13. EMG не питает силовые AUX-нагрузки.
14. ИЗС — 6 каналов по 30 Вт.
15. ИЗС питаются через две группы boost 12→32 В.
16. Каждый ИЗС-канал управляется отдельно.
17. Диммирование ИЗС — от 5%.
18. Базовый внешний интерфейс — RS-485.
19. Сервисные интерфейсы — UART/SWD.
20. Следующий рубеж — schematic freeze V1.4.

## 4. Последствия

Положительные последствия:

1. появляется единая ссылка на проект;
2. требования, архитектура, сигналы, аварии, PCB, BOM и тесты разделены по файлам;
3. открытые вопросы вынесены отдельно;
4. дальнейшая схема должна соответствовать зафиксированной логике;
5. появляется дорожная карта до рабочего изделия.

Ограничения:

1. нельзя переходить к PCB до закрытия вопросов, влияющих на схему;
2. нельзя менять принцип EMG без отдельного решения;
3. нельзя питать ИЗС от EMG;
4. нельзя убирать precharge без отдельного решения;
5. нельзя заменять RS-485 как базовый интерфейс без отдельного решения;
6. нельзя считать BOM финальным до выбора конкретных компонентов и испытаний.

## 5. Проверка решения

Решение считается корректным, если:

1. все следующие работы ссылаются на `PROJECT_MASTER.md`;
2. изменения архитектуры фиксируются в `ARCHITECTURE_BASELINE.md` и ADR;
3. открытые вопросы ведутся в `OPEN_QUESTIONS.md`;
4. хронология обновляется при каждом существенном шаге;
5. тесты соответствуют требованиям и состояниям;
6. схема не противоречит запрету питания силовых нагрузок от EMG.
