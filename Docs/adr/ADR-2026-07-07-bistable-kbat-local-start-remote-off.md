# ADR-2026-07-07: Бистабильные K_BATx, отдельный LOCAL_START и REMOTE_OFF на контакте 11

Статус:

```text
SUPERSEDED
```

Заменяющий документ:

```text
ADR-2026-07-10-monostable-kbat-self-hold.md
```

## 1. Причина замены

Решение о бистабильном контакторе было основано на требовании исключить постоянное питание катушки. После уточнения фактического требования безопасности установлено, что при исчезновении защищённого выхода BMS контактор обязан автоматически вернуться в `OPEN`, а после восстановления BMS не должен включаться без нового ручного `LOCAL_START`.

Бистабильный контактор сохраняет механическое состояние при снятии напряжения и не обеспечивает это поведение без отдельной энергии и команды RESET. Поэтому он больше не является рабочей архитектурой проекта.

## 2. Отменённые положения

Отменены:

1. бистабильное сохранение положения `K_BATx`;
2. две катушки SET/RESET;
3. одна реверсивная катушка;
4. отдельный импульс включения SET;
5. отдельный импульс выключения RESET;
6. one-shot-формирователь RESET;
7. требование отсутствия постоянного питания катушки;
8. сохранение `CLOSED` при исчезновении защищённого выхода BMS.

## 3. Что сохраняет силу

1. `K_BAT1/K_BAT2` остаются однополюсными контакторами по положительному проводнику.
2. Контакты 1–5 `СН-176А-12` используются для `BAT+`.
3. Контакты 6–10 используются для `BAT−`.
4. Контакт 11 сохраняет имя `REMOTE_OFF`.
5. Контакт 12 остаётся `RESERVE`.
6. `LOCAL_START` остаётся отдельным двухконтактным разъёмом.
7. В корпусах АКБ нет дополнительной активной электроники и цифровых интерфейсов.
8. BMS остаётся автономной и не взаимодействует с управляющей системой.

## 4. Действующая архитектура

Действующая архитектура определена в `ADR-2026-07-10-monostable-kbat-self-hold.md`:

```text
моностабильный SPST-NO контактор
+ одна continuous-duty катушка
+ вспомогательный NO-контакт самоподхвата
+ LOCAL_START как кратковременное замыкание
+ REMOTE_OFF как разрыв цепи удержания
+ автоматический OPEN при исчезновении BAT_PROT+/BAT_PROT−
+ ручной повторный запуск после восстановления BMS
```
