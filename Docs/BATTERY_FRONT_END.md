# Battery Front-End АКБ_1 / АКБ_2

## 1. Назначение

Battery Front-End обеспечивает безопасное подключение каждой основной АКБ к общей шине `PACK_BUS`, измерение параметров батарейной ветви, управляемую параллельную работу в режиме `RUN`, контролируемую межблочную балансировку в режиме `DECK_BALANCE` и аварийное электронное отключение силового тракта.

Численные требования к механическому контактору и цепи катушки приведены в `Docs/KBAT_ELECTRICAL_REQUIREMENTS.md`.

## 2. Симметричная архитектура двух ветвей

```text
АКБ_1 → autonomous BMS → protected output → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                                                       ├→ PACK_BUS
АКБ_2 → autonomous BMS → protected output → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘
```

Линии АКБ_1 и АКБ_2 одинаковы по типу соединителя, длине, распайке, сечению жил и силовой топологии Battery Front-End.

## 3. Минимальный состав корпуса АКБ

Внутри каждого корпуса основной АКБ находятся только:

1. аккумуляторная сборка LiFePO4 4s24p, 12,8 В, 144 А·ч;
2. существующая автономная BMS;
3. батарейный силовой предохранитель;
4. однополюсный моностабильный нормально разомкнутый контактор `K_BATx`;
5. одна катушка непрерывного режима;
6. вспомогательный NO-контакт электрического самоподхвата;
7. защита цепи катушки `F_CTRL`;
8. минимальная пассивная цепь подавления выброса;
9. необходимая проводка.

На корпусе установлены `СН-176А-12` и отдельный двухконтактный разъём `LOCAL_START`.

В корпусе АКБ не устанавливаются отдельный MCU, дополнительная управляющая или интерфейсная плата, RS-485/CAN/UART, дополнительный ADC, отдельный DC/DC, цифровая телеметрия BMS или raw-ветвь от ячеек.

BMS работает автономно. Проект использует только внешние защищённые `BAT_PROT+ / BAT_PROT−` и не зависит от внутренней топологии BMS.

## 4. Межкорпусные разъёмы

### СН-176А-12

| Контакты | Назначение | Статус |
|---|---|---|
| 1–5 | BAT+ | принято как рабочая группа |
| 6–10 | BAT− | принято как рабочая группа |
| 11 | REMOTE_OFF / K_BAT_HOLD_RETURN | возвратная цепь катушки |
| 12 | RESERVE | не назначен |

Предварительная нагрузка pin 11:

```text
≤0,25 А continuous
≤2,0 А / 150 мс pulsed
```

RS-485, CAN, UART и цифровая телеметрия BMS через СН-176А-12 не передаются.

### LOCAL_START

Отдельный двухконтактный разъём используется для кратковременного ручного замыкания цепи запуска.

`LOCAL_START` не проходит через СН-176А-12, должен быть нормально разомкнутым, пружинно-возвратным и без фиксации. Его предварительный рейтинг не ниже 0,5 А при 16 В DC и 2 А / 150 мс.

## 5. Поведение K_BATx

```text
LOCAL_START short
→ coil energized
→ K_BATx CLOSED
→ K_BAT_AUX_NO CLOSED
→ electrical self-hold

REMOTE_OFF loop open
или BMS protected output lost
→ coil de-energized
→ K_BATx OPEN
→ K_BAT_AUX_NO OPEN
```

После восстановления BMS или REMOTE_OFF loop контактор остаётся `OPEN` до нового `LOCAL_START`.

Катушка питается только от защищённого выхода собственной АКБ. Предпочтителен встроенный economizer.

Предварительная расчётная рамка катушки:

| Параметр | Требование |
|---|---:|
| Input withstand | 0…16,0 В DC |
| Guaranteed pickup | 9,0…16,0 В |
| Guaranteed hold | 7,5…16,0 В |
| Inrush | ≤2,0 А / ≤150 мс |
| Hold current | target ≤0,15 А; limit ≤0,25 А |
| Hold power | target ≤2 Вт; limit ≤4 Вт |

## 6. Состав Battery Front-End в корпусе электроники

1. входная защита после СН-176А-12;
2. TVS/EMI-защита батарейных входов;
3. измерение напряжения батарейной ветви;
4. измерение тока батарейной ветви;
5. определение наличия батарейной линии;
6. дистанционный размыкающий элемент `REMOTE_OFF_NC` для цепи контакта 11;
7. аппаратный путь `EXT_KILL` к разрыву обеих цепей удержания;
8. управляемый двунаправленный основной силовой тракт `MAIN_SWx`;
9. ограниченный путь `BALANCE_SWx + R_BALx`;
10. быстродействующая защита от короткого замыкания и перегрузки;
11. контроль напряжения и тока после команды выключения контактора.

Отдельный one-shot-драйвер RESET отсутствует.

Предварительные требования к `REMOTE_OFF_NC`:

| Параметр | Требование |
|---|---:|
| DC rating | ≥30 В DC, ≥0,5 А continuous, ≥2 А pulsed |
| Semiconductor blocking rating | ≥60 В |
| Closed-state drop | ≤0,2 В при hold current |
| Open-state leakage | ≤1 мА |
| OFF duration | `≥ max(250 мс; 5 × t_release_max)` |
| System release target | ≤100 мс |

Конкретный способ реализации `REMOTE_OFF_NC` и `EXT_KILL_NC` определяется после выбора кандидата катушки и расчёта suppression.

## 7. Suppression

1. Простой flyback-диод не является вариантом по умолчанию.
2. Анализируются TVS, bidirectional TVS и diode+resistor.
3. Предварительный диапазон clamp — 24…36 В.
4. Clamp не должен превышать рабочий рейтинг размыкающего элемента с запасом не менее 20 %.
5. Фактическая энергия рассчитывается по измеренным L и I катушки.
6. Вариант с системным release time более 100 мс отклоняется.

## 8. Контроль контактора без локальной платы

Контроль результата выполняется по совокупности признаков:

```text
факт размыкания REMOTE_OFF loop
+
выдержка нормативного t_OFF
+
время отпускания
+
BATx_RAW_V
+
BATx_I
+
PACK_BUS_V
```

Контакт 12 не используется для feedback.

## 9. Силовые требования к K_BATx

| Параметр | Предварительное требование |
|---|---:|
| Main continuous current | минимум 30 А; целевой класс 50 А и выше |
| DC breaking capacity | минимум 30 А при 16 В DC |
| Initial contact resistance | ≤2 мОм |
| Post-life resistance | ≤5 мОм |
| Mechanical life | ≥100 000 циклов |
| Electrical life | ≥10 000 операций на подтверждённом токе |
| Operating temperature | не уже −20…+60 °C |

`K_BATx` не заменяет fuse и `MAIN_SWx` как защиту от короткого замыкания.

## 10. Режим RUN

Обе исправные основные АКБ работают параллельно. MCU измеряет ток и напряжение каждой ветви, но не отключает исправную батарею только из-за различия уровня заряда.

Различия напряжений, токов и расчётного SoC используются для контроля и диагностики состояния батарейных ветвей.

## 11. Режим DECK_BALANCE

В `DECK_BALANCE` тяжёлые нагрузки отключены, а межблочный ток ограничивается трактом `BALANCE_SWx + R_BALx` в корпусе электроники.

BMS внутри каждого блока продолжает выполнять собственную балансировку ячеек независимо от MCU аппарата.

## 12. HARD_OFF

Штатное физическое отключение выполняется размыканием цепей `REMOTE_OFF` для `K_BAT1` и `K_BAT2` после сохранения журнала и снятия тяжёлых нагрузок. Когда возможно, `MAIN_SWx` предварительно снижает ток перед механическим размыканием.

При аварийном отключении `EXT_KILL` аппаратно размыкает обе цепи удержания и отключает `MAIN_SW1/MAIN_SW2` независимо от MCU.

После отключения основных АКБ MCU сохраняет причину аварии от EMG.

## 13. Работа при защите BMS

```text
BMS output disabled
→ coil current = 0
→ K_BATx OPEN

BMS output restored
→ K_BAT_AUX_NO remains OPEN
→ no automatic restart
```

Сценарий испытывается для фактической BMS и любой будущей BMS, сохраняющей внешний защищённый двухполюсный выход.
