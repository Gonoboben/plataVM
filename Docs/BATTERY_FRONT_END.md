# Battery Front-End АКБ_1 / АКБ_2

## 1. Назначение

Battery Front-End обеспечивает безопасное подключение каждой основной АКБ к общей шине `PACK_BUS`, измерение параметров батарейной ветви, управляемую параллельную работу в режиме `RUN`, контролируемую межблочную балансировку в режиме `DECK_BALANCE` и аварийное электронное отключение силового тракта.

## 2. Симметричная архитектура двух ветвей

```text
АКБ_1 → autonomous BMS → protected output → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                                                      ├→ PACK_BUS
АКБ_2 → autonomous BMS → protected output → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘
```

Линии АКБ_1 и АКБ_2 должны быть одинаковыми по типу соединителя, длине, распайке, сечению жил и силовой топологии Battery Front-End.

## 3. Минимальный состав корпуса АКБ

Внутри каждого корпуса основной АКБ находятся только:

1. аккумуляторная сборка LiFePO4 4s24p, 12,8 В, 144 А·ч;
2. существующая автономная BMS для защиты и балансировки ячеек;
3. батарейный предохранитель;
4. однополюсный моностабильный нормально разомкнутый контактор `K_BATx` в положительном проводнике;
5. одна катушка непрерывного режима;
6. вспомогательный NO-контакт электрического самоподхвата;
7. защита цепи катушки `F_CTRL`;
8. минимальная пассивная цепь подавления выброса;
9. необходимая проводка.

На корпусе установлены:

1. `СН-176А-12`;
2. отдельный двухконтактный разъём `LOCAL_START`.

В корпусе АКБ не устанавливаются:

1. отдельный MCU;
2. дополнительная управляющая плата;
3. RS-485, CAN или UART-трансивер;
4. локальная интерфейсная плата;
5. дополнительный ADC;
6. отдельный DC/DC для управляющей электроники;
7. цифровая телеметрия BMS;
8. raw-ветвь от ячеек.

BMS работает автономно. Проект использует только внешние защищённые `BAT_PROT+ / BAT_PROT−` и не зависит от внутренней топологии BMS.

## 4. Межкорпусные разъёмы

### СН-176А-12

| Контакты | Назначение | Статус |
|---|---|---|
| 1–5 | BAT+ | принято как рабочая группа |
| 6–10 | BAT− | принято как рабочая группа |
| 11 | REMOTE_OFF / K_BAT_HOLD_RETURN | возвратная цепь катушки |
| 12 | RESERVE | не назначен |

RS-485, CAN, UART и цифровая телеметрия BMS через СН-176А-12 не передаются.

### LOCAL_START

Отдельный двухконтактный разъём используется для кратковременного ручного замыкания цепи запуска.

`LOCAL_START` не проходит через СН-176А-12, должен быть нормально разомкнутым, пружинно-возвратным и без фиксации.

## 5. Поведение K_BATx

```text
LOCAL_START short
→ coil energized
→ K_BATx CLOSED
→ K_BAT_AUX_NO CLOSED
→ electrical self-hold

REMOTE_OFF loop open
или BMS protected output lost
→ coil de-energized
→ K_BATx OPEN
→ K_BAT_AUX_NO OPEN
```

После восстановления BMS или REMOTE_OFF loop контактор остаётся `OPEN` до нового `LOCAL_START`.

Катушка питается только от защищённого выхода собственной АКБ. Предпочтителен встроенный economizer, но конкретное исполнение пока не выбрано.

## 6. Состав Battery Front-End в корпусе электроники

1. входная защита после СН-176А-12;
2. TVS/EMI-защита батарейных входов;
3. измерение напряжения батарейной ветви;
4. измерение тока батарейной ветви;
5. определение наличия батарейной линии;
6. дистанционный размыкающий элемент `REMOTE_OFF_NC` для цепи контакта 11;
7. аппаратный путь `EXT_KILL` к разрыву обеих цепей удержания;
8. управляемый двунаправленный основной силовой тракт `MAIN_SWx`;
9. ограниченный путь `BALANCE_SWx + R_BALx`;
10. быстродействующая защита от короткого замыкания и перегрузки;
11. контроль напряжения и тока после команды выключения контактора.

Отдельный one-shot-драйвер RESET отсутствует. Конкретная реализация дистанционного размыкающего элемента определяется после задания тока катушки, DC-рейтинга и минимального времени разрыва.

## 7. Контроль контактора без локальной платы

Контроль результата выполняется по совокупности признаков:

```text
факт размыкания REMOTE_OFF loop
+
время отпускания
+
напряжение BATx_RAW
+
ток BATx_I
+
состояние PACK_BUS
```

Контакт 12 не используется для feedback.

## 8. Режим RUN

Обе исправные основные АКБ работают параллельно. MCU измеряет ток и напряжение каждой ветви, но не отключает исправную батарею только из-за различия уровня заряда.

Различия напряжений, токов и расчётного SoC используются для контроля и диагностики состояния батарейных ветвей.

## 9. Режим DECK_BALANCE

В `DECK_BALANCE` тяжёлые нагрузки отключены, а межблочный ток ограничивается трактом `BALANCE_SWx + R_BALx` в корпусе электроники.

BMS внутри каждого блока продолжает выполнять собственную балансировку ячеек независимо от MCU аппарата.

## 10. HARD_OFF

Штатное физическое отключение выполняется размыканием цепей `REMOTE_OFF` для `K_BAT1` и `K_BAT2` после сохранения журнала и снятия тяжёлых нагрузок.

При аварийном отключении `EXT_KILL` аппаратно размыкает обе цепи удержания и отключает `MAIN_SW1/MAIN_SW2` независимо от MCU.

После отключения основных АКБ MCU сохраняет причину аварии от EMG.

## 11. Работа при защите BMS

```text
BMS output disabled
→ coil current = 0
→ K_BATx OPEN

BMS output restored
→ K_BAT_AUX_NO remains OPEN
→ no automatic restart
```

Этот сценарий должен быть испытан для текущей BMS с коммутацией отрицательного проводника и для любой будущей BMS, сохраняющей внешний защищённый двухполюсный выход.
