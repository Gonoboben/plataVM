# Предложение по электрической реализации LOCAL_START и REMOTE_OFF

Дата: 2026-07-07  
Статус:

```text
PROPOSAL — не является принятой архитектурой до отдельного подтверждения
```

## 1. Назначение

Документ предлагает электрическую реализацию двух команд управления `K_BAT1/K_BAT2` при неизменных внешних ограничениях:

1. `LOCAL_START` — отдельный двухконтактный разъём на корпусе каждой АКБ;
2. `REMOTE_OFF` — только контакт 11 `СН-176А-12`;
3. контакт 12 остаётся `RESERVE`;
4. энергия переключения контактора поступает от собственной АКБ;
5. в корпусе АКБ не устанавливается дополнительная активная электроника;
6. постоянное питание катушки запрещено;
7. `EXT_KILL` должен формировать выключение независимо от MCU.

Конкретная модель контактора в этом документе не выбирается.

## 2. Основной вывод

Для действующей распайки предпочтительна и фактически требуется конструкция с двумя независимыми катушками:

```text
SET coil   — включение через LOCAL_START
RESET coil — выключение через REMOTE_OFF
```

Однокатушечная реверсивная схема не принимается как базовая, потому что для неё необходимо менять полярность между двумя выводами катушки. При одном удалённом проводнике `REMOTE_OFF`, резервном контакте 12 и запрете H-моста внутри корпуса АКБ надёжное независимое формирование SET и RESET невозможно без изменения принятой архитектуры.

## 3. Функциональная схема корпуса АКБ

Точные полярности и точки подключения зависят от топологии существующей BMS. Функционально схема должна иметь следующий вид:

```text
BMS_PROTECTED_SUPPLY
    │
    ├─ F_SET / passive current protection
    │      │
    │      └─ AUX_OPEN_STATE ─ SET_COIL ─ LS1
    │                                      │
    │                              external momentary short
    │                                      │
CTRL_RETURN ─────────────────────────────── LS2

RAW_PACK_RESET_SUPPLY
    │
    ├─ F_RESET / passive current protection
    │      │
    │      └─ AUX_CLOSED_STATE ─ RESET_COIL ─ pin 11 REMOTE_OFF
    │
CTRL_RETURN ────────────────────────────────────────────────
```

Обозначения:

1. `AUX_OPEN_STATE` замкнут только тогда, когда главный силовой контакт `K_BATx` разомкнут;
2. `AUX_CLOSED_STATE` замкнут только тогда, когда главный силовой контакт `K_BATx` замкнут;
3. оба состояния могут быть реализованы одним механически связанным перекидным вспомогательным контактом;
4. `F_SET` и `F_RESET` — маломощные защитные элементы ветвей катушек, тип определяется после выбора катушки;
5. на каждой катушке применяется индивидуальная пассивная цепь ограничения выброса напряжения;
6. активных формирователей импульса внутри корпуса АКБ нет.

## 4. Электрическая реализация LOCAL_START

### 4.1 Рабочая цепь

```text
BMS_PROTECTED_SUPPLY
→ F_SET
→ AUX_OPEN_STATE
→ SET_COIL
→ контакт LS1
→ кратковременное внешнее замыкание LS1–LS2
→ CTRL_RETURN
```

### 4.2 Последовательность включения

```text
оператор кратковременно замыкает LOCAL_START
→ SET_COIL получает ток
→ K_BATx переходит в CLOSED
→ вспомогательный контакт покидает AUX_OPEN_STATE
→ цепь SET_COIL аппаратно размыкается
→ K_BATx остаётся CLOSED без питания катушки
```

### 4.3 Требования

1. `LOCAL_START` несёт полный ток катушки SET.
2. Разъём, внешняя кнопка и проводка должны быть рассчитаны на ток катушки и её индуктивный характер.
3. Внешний орган управления должен быть пружинно-возвратным, без фиксации.
4. После успешного перехода в CLOSED ток SET должен прекратиться вспомогательным контактом даже при удержании кнопки.
5. При механическом отказе контактора длительный ток ограничивается или отключается пассивной защитой ветви `F_SET`.
6. Повторное замыкание `LOCAL_START` при уже закрытом контакторе не должно подавать ток на SET, поскольку `AUX_OPEN_STATE` уже разомкнут.
7. Полярность двух контактов `LOCAL_START` становится фиксированной и должна быть указана на схеме и маркировке кабельной оснастки.

## 5. Электрическая реализация REMOTE_OFF

### 5.1 Цепь внутри корпуса АКБ

```text
RAW_PACK_RESET_SUPPLY
→ F_RESET
→ AUX_CLOSED_STATE
→ RESET_COIL
→ контакт 11 СН-176А-12
```

### 5.2 Цепь внутри корпуса электроники

```text
BATx_REMOTE_OFF_REQ ─┐
                     ├→ hardware OR / trigger
EXT_KILL ────────────┘
→ non-retriggerable one-shot, питание от 5V_CRIT / EMG
→ protected low-side switch Q_OFFx
→ BAT− / CTRL_RETURN
```

Физический ток:

```text
собственная АКБ
→ RESET_COIL
→ контакт 11
→ Q_OFFx
→ BAT− собственной АКБ
```

Корпус электроники не питает катушку своей силовой шиной, а только замыкает её токовый путь на общий возврат.

### 5.3 Последовательность выключения

```text
MCU или EXT_KILL запускает аппаратный one-shot
→ Q_OFFx замыкает контакт 11 на CTRL_RETURN на конечное время t_OFF
→ RESET_COIL получает ток
→ K_BATx переходит в OPEN
→ вспомогательный контакт покидает AUX_CLOSED_STATE
→ ток RESET аппаратно прекращается
→ one-shot завершает импульс независимо от состояния входного запроса
```

### 5.4 Требования к one-shot

1. Отдельный канал для АКБ_1 и отдельный канал для АКБ_2.
2. Питание формирователей от `5V_CRIT / EMG`.
3. Запуск от MCU и от аппаратного `EXT_KILL`.
4. `EXT_KILL` не проходит через MCU.
5. Импульс конечный и non-retriggerable.
6. Удержание `EXT_KILL` или программного запроса не продлевает импульс бесконечно.
7. Повторный импульс допускается только после возврата входного запроса в неактивное состояние и повторного фронта.
8. Длительность `t_OFF` выбирается после определения времени срабатывания RESET-катушки и её максимально допустимой длительности питания.
9. Q_OFFx должен выдерживать максимальный ток катушки, индуктивный выброс, обрыв линии и ошибочное внешнее напряжение на контакте 11.
10. Пассивное ограничение выброса должно обеспечивать достаточно быстрое спадание тока и не препятствовать завершению механического переключения.

## 6. Питание SET и RESET относительно BMS

Рекомендуемое функциональное разделение:

1. SET получает энергию только с защищённой стороны BMS.
2. RESET получает энергию от собственной АКБ независимо от состояния главного силового контакта и, по возможности, независимо от силового ключа BMS.
3. RESET-ветвь имеет отдельную маломощную защиту и не связана с внешней нагрузкой.

Причины:

1. при отключённой BMS команда LOCAL_START не должна заранее замыкать контактор и создавать автоматическое подключение нагрузки после самовосстановления BMS;
2. команда REMOTE_OFF должна оставаться доступной после отключения основной силовой ветви;
3. RESET-ветвь не может включить внешнюю нагрузку и используется только для физического размыкания контактора.

Точная точка отбора `RAW_PACK_RESET_SUPPLY` не фиксируется до изучения схемы существующей BMS. Если BMS коммутирует отрицательный проводник, функциональная схема должна быть адаптирована без изменения логики SET/RESET.

## 7. Реакция на отказы

| Отказ или состояние | Ожидаемое поведение |
|---|---|
| LOCAL_START удерживается после успешного включения | AUX_OPEN_STATE размыкает SET_COIL; длительного тока нет |
| LOCAL_START удерживается, но контактор заклинил | F_SET/пассивная защита ограничивает или отключает длительный ток; фиксируется отказ включения |
| LOCAL_START повторно замкнут при CLOSED | SET_COIL не питается, потому что AUX_OPEN_STATE разомкнут |
| MCU удерживает REMOTE_OFF_REQ | one-shot выдаёт только один конечный импульс |
| EXT_KILL удерживается | каждый канал выдаёт один конечный импульс до отпускания и нового фронта |
| контакт 11 замкнут на BAT− при CLOSED | контактор переходит в OPEN, затем AUX_CLOSED_STATE разрывает ток RESET |
| контакт 11 замкнут на BAT− при OPEN | RESET_COIL обесточена через AUX_CLOSED_STATE |
| контакт 11 оборван | механическое REMOTE_OFF невозможно; MAIN_SWx отключается, формируется fault по отсутствию ожидаемого изменения BATx_I/PACK_BUS_V |
| Q_OFFx пробит | поведение аналогично замыканию контакта 11 на BAT−; после открытия контактора вспомогательный контакт прекращает ток |
| Q_OFFx не открывается | REMOTE_OFF не выполняется; MAIN_SWx остаётся независимым резервным отключением |
| вспомогательный контакт не размыкает катушку | one-shot ограничивает RESET; пассивная защита ограничивает SET |
| K_BATx сварен или механически заклинил | MAIN_SWx снимает ток ветви, MCU фиксирует отсутствие физического отключения |
| BMS отключила основную силовую ветвь | SET блокирован; RESET должен оставаться доступным от защищённой маломощной raw-ветви |

## 8. Почему не используется пассивный RC-формирователь как основной вариант

Серийный конденсатор может сформировать конечный импульс без активной электроники, но его ёмкость и энергия напрямую зависят от сопротивления катушки, минимального тока срабатывания и требуемой длительности. До выбора контактора такой узел нельзя корректно рассчитать. Для силовой катушки он может потребовать крупный накопительный конденсатор и создаёт дополнительный элемент с ограниченным ресурсом внутри герметичного корпуса.

Поэтому RC-формирователь не принимается как базовый вариант. Основной способ прекращения тока внутри корпуса АКБ — механически связанный вспомогательный контакт, а для REMOTE_OFF дополнительно применяется аппаратный one-shot в корпусе электроники.

## 9. Решения, которые предлагается принять

1. Зафиксировать двухкатушечный бистабильный контактор `SET/RESET`.
2. Исключить однокатушечную реверсивную схему из текущей архитектуры.
3. Реализовать `LOCAL_START` как прямое замыкание цепи SET-катушки.
4. Реализовать `REMOTE_OFF` как low-side коммутацию RESET-катушки через контакт 11.
5. Требовать механически связанный перекидной вспомогательный контакт для самопрерывания SET и RESET.
6. Реализовать два независимых non-retriggerable one-shot в корпусе электроники.
7. Питать аппаратные one-shot от `5V_CRIT / EMG`.
8. Сохранить контакт 12 без назначения.
9. Не добавлять активную электронику в корпуса АКБ.
10. Проверить топологию существующей BMS до фиксации точек отбора SET- и RESET-питания.

## 10. Что остаётся открытым после принятия принципа

1. модель и схема существующей BMS;
2. номинальное и допустимое напряжение SET/RESET-катушек;
3. ток катушек при минимальном и максимальном напряжении АКБ;
4. минимальная и максимальная длительность импульса;
5. тип и номинал F_SET/F_RESET или PTC;
6. тип цепи подавления выброса каждой катушки;
7. рейтинг контактов LOCAL_START;
8. параметры Q_OFFx и его защиты;
9. номинальный и предельный силовой ток K_BATx;
10. DC breaking capacity;
11. температура, вибрация и ресурс контактора;
12. поведение контактора при недостаточной энергии импульса.

До закрытия этих параметров конкретная модель контактора не выбирается.
