# Дерево питания ПДУ БНПА V1.5

## 1. Схема верхнего уровня

```text
АКБ_1 → BMS → fuse → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                                      ├→ PACK_BUS
АКБ_2 → BMS → fuse → K_BAT2 → СН-176А-12 → BFE_2 ┘

PACK_BUS
    ├─ POWER_12V_BUS → CH1...CH14
    ├─ 5V_SYS_BUS → 5V_OUT1...5V_OUT10
    ├─ LIGHT_POWER_BRANCH → LED drivers → 6 × Epistar XY-J45
    └─ RESERVE_BRANCH → EMG charge path / KEEP_ALIVE / 5V_CRIT / 3V3_CRIT
```

## 2. Главная силовая шина

`PACK_BUS` является единственной главной силовой шиной после двух управляемых Battery Front-End.

Отдельные `K_MAIN` и `MAIN_INPUT_BUS` отсутствуют.

## 3. Батарейные ветви

Обе батарейные ветви симметричны и в режиме `RUN` работают параллельно.

Каждая ветвь включает:

1. аккумуляторную сборку LiFePO4 4s24p, 12,8 В, 144 А·ч;
2. существующую BMS в корпусе АКБ;
3. предохранитель;
4. однополюсный бистабильный контактор `K_BATx` по плюсу;
5. межкорпусную линию `СН-176А-12`, длина 1 м;
6. Battery Front-End в корпусе электроники с измерением напряжения и тока;
7. основной электронный силовой тракт;
8. ограниченный тракт `DECK_BALANCE`.

Температура ячеек и цифровой статус BMS через батарейную линию не передаются.

## 4. Управление контактором

Контактор питается непосредственно от собственной АКБ и сохраняет состояние после кратковременного импульса.

```text
LOCAL_START, отдельный 2-pin разъём
→ кратковременный импульс включения
→ K_BATx остаётся замкнутым

REMOTE_OFF, контакт 11 СН-176А-12
→ кратковременный импульс выключения
→ K_BATx остаётся разомкнутым
```

Контакт 12 `СН-176А-12` — резерв.

Постоянное питание катушки отсутствует. Моностабильный контактор не входит в текущую архитектуру.

## 5. Правило разделения ветвей нагрузки

`POWER_12V_BUS`, `5V_SYS_BUS`, `LIGHT_POWER_BRANCH` и `RESERVE_BRANCH` являются отдельными ветвями от `PACK_BUS`.

## 6. Что питается от резерва

Только critical/keep-alive домен:

1. MCU;
2. связь;
3. критические датчики;
4. fault/event logic;
5. 5V_CRIT;
6. 3V3_CRIT.

## 7. Что не питается от резерва

1. CH1...CH14;
2. 5V_SYS_BUS;
3. LED-драйверы;
4. силовые внешние нагрузки.

## 8. HARD_OFF

```text
штатно:
снять тяжёлые нагрузки
→ сохранить журнал и SoC
→ импульс REMOTE_OFF для K_BAT1
→ импульс REMOTE_OFF для K_BAT2
→ проверить токи ветвей
→ проверить разряд PACK_BUS
→ завершить работу MCU от EMG

аварийно:
EXT_KILL
→ аппаратно отключить электронные силовые тракты
→ аппаратно сформировать REMOTE_OFF для K_BAT1
→ аппаратно сформировать REMOTE_OFF для K_BAT2
→ записать причину от EMG
→ проверить разряд PACK_BUS
```

## 9. Следующий этап

До выбора модели контактора должны быть определены электрические реализации `LOCAL_START` и `REMOTE_OFF`, после чего формируются требования к катушке, импульсам, силовому току, DC breaking capacity и отказобезопасности.
