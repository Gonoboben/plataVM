# Дерево питания ПДУ БНПА V1.5

## 1. Схема верхнего уровня

```text
АКБ_1 → K_BAT1 → СН-176А-12 → BFE_1 ┐
                                      ├→ PACK_BUS
АКБ_2 → K_BAT2 → СН-176А-12 → BFE_2 ┘

PACK_BUS
    ├─ POWER_12V_BUS → CH1...CH14
    ├─ 5V_SYS_BUS → 5V_OUT1...5V_OUT10
    ├─ LIGHT_POWER_BRANCH → LED drivers → 6 × Epistar XY-J45
    └─ RESERVE_BRANCH → EMG charge path / KEEP_ALIVE / 5V_CRIT / 3V3_CRIT
```

## 2. Главная силовая шина

`PACK_BUS` является единственной главной силовой шиной после двух управляемых Battery Front-End.

Отдельные `K_MAIN` и `MAIN_INPUT_BUS` не используются.

## 3. Батарейные ветви

Обе батарейные ветви симметричны и в режиме `RUN` работают параллельно.

Каждая ветвь включает:

1. BMS в корпусе АКБ;
2. предохранитель;
3. однополюсный контактор `K_BATx` по плюсу;
4. межкорпусную линию `СН-176А-12`, длина 1 м;
5. измерения напряжения, тока и температуры;
6. основной электронный силовой тракт;
7. ограниченный тракт мягкого подключения и `DECK_BALANCE`.

## 4. Правило разделения ветвей нагрузки

`POWER_12V_BUS`, `5V_SYS_BUS`, `LIGHT_POWER_BRANCH` и `RESERVE_BRANCH` являются отдельными ветвями от `PACK_BUS`.

## 5. Что питается от резерва

Только critical/keep-alive домен:

1. MCU;
2. связь;
3. критические датчики;
4. fault/event logic;
5. 5V_CRIT;
6. 3V3_CRIT.

## 6. Что не питается от резерва

1. CH1...CH14;
2. 5V_SYS_BUS;
3. LED-драйверы;
4. силовые внешние нагрузки.

## 7. HARD_OFF

```text
штатно:
снять тяжёлые нагрузки → сохранить журнал → K_BAT1 OFF → K_BAT2 OFF

аварийно:
отключить электронные силовые тракты → K_BAT1 OFF → K_BAT2 OFF → записать причину от EMG
```

После отключения контролируется разряд `PACK_BUS`.
