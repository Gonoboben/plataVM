## 2. Q-MECH-006 — доступный внутренний объём

Ответ владельца:

```text
Внутренние размеры: цилиндр Ø130 мм × 1000 мм.
Один торец заварен, второй закрывается крышкой с разъёмами.
Дополнительные сведения о стенке и материале для проектирования платы не требуются.
```

Инженерная фиксация:

1. `Ø130 мм × 1000 мм` принимается как заданный владельцем доступный внутренний envelope;
2. проект PCB не выполняет расчёт pressure hull, материала или толщины стенки;
3. геометрические допуски и пригодность корпуса остаются ответственностью владельца/изготовителя корпуса;
4. электронная сборка должна укладываться в более жёсткий рабочий envelope из Q-MECH-007.

Статус:

```text
CLOSED_OWNER_DEFINED_ENVELOPE
```

## 3. Q-MECH-007 — envelope электронной сборки

Ответ владельца:

```text
Ширина не более 100 мм.
Многоуровневая конструкция.
Предварительная длина около 250 мм.
Высота 80 мм.
Карта разъёмов крышки для текущей задачи не требуется.
```

Принято:

| Параметр | Ограничение |
|---|---:|
| Максимальная ширина PCB/сборки | 100 мм |
| Предварительная максимальная длина электронной сборки | 250 мм |
| Максимальная высота многоуровневой сборки | 80 мм |
| Число уровней | определяется компоновкой внутри 80 мм |
| Карта разъёмов крышки | owner-controlled, вне текущего PCB scope |

Предел 250 мм имеет статус предварительного целевого envelope, а не окончательного чертежного допуска.

Статус:

```text
CLOSED_OWNER_ENVELOPE_PRELIMINARY_LENGTH
```

## 4. Q-MECH-008 — способ установки и извлечения

Ответ владельца:

```text
Вся электроника извлекается вместе с крышкой.
Платы крепятся несколькими винтами к стойкам.
Винты и стойки владелец подбирает самостоятельно.
В PCB необходимо предусмотреть крепёжные отверстия.
```

Принято:

1. отдельный обязательный продольный carrier/tray не является частью текущего проекта;
2. электронная сборка механически связана с крышкой и извлекается как единый модуль;
3. платы устанавливаются многоуровнево на стойках;
4. модель, материал и длина стоек, а также винты находятся в owner-controlled mechanical scope;
5. в каждой PCB предусматриваются достаточные крепёжные отверстия и keepout вокруг них;
6. тяжёлые компоненты и силовые шины не должны удерживаться только пайкой.

Статус:

```text
CLOSED_OWNER_MECHANICAL_CONCEPT
```

## 5. Q-MECH-009 — pressure/vibration scope и фиксация компонентов

Ответ владельца:

```text
Pressure-hull characteristics не требуются для задачи проектирования плат.
Точный vibration profile не задан.
Компоненты припаяны или привинчены; дополнительная фиксация выполняется полиэтиленовым hot-melt adhesive.
```

Инженерная интерпретация:

1. прочность герметичного цилиндра, крышки и уплотнения находится вне scope проекта электроники;
2. проект принимает корпус как owner-provided qualified enclosure;
3. герметичность сама по себе не устраняет внешнее дифференциальное давление, поэтому соответствующая механическая ответственность явно остаётся у владельца/изготовителя корпуса;
4. отсутствие vibration profile не блокирует Architecture A/B, но требует конструктивной фиксации тяжёлых элементов;
5. пайка и винтовое крепление являются основными способами удержания;
6. полиэтиленовый hot-melt adhesive допускается только как вспомогательная anti-vibration/strain-relief фиксация;
7. hot-melt adhesive не должен быть единственной опорой тяжёлого компонента, наноситься на горячие детали или препятствовать ремонту/электроизоляционным зазорам;
8. совместимость клея с диапазоном −20…+60 °C, conformal coating и материалами проверяется на образце.

Статус:

```text
CLOSED_OWNER_SCOPE_WITH_DFM_RULES
```

## 6. Q-MECH-010 — тепловая архитектура

Ответ владельца:

```text
Тепловой контакт электронной сборки с корпусом запрещён.
Работа — только за счёт внутренней естественной конвекции.
Компоненты должны иметь малое тепловыделение.
```

Принято:

1. carrier, радиаторы, thermal pads и PCB не должны использовать корпус как штатный теплоотвод;
2. охлаждение — пассивное, внутренней естественной конвекцией и теплопроводностью внутри самой PCB/сборки;
3. приоритет имеют высокоэффективные топологии, низкие потери проводимости/переключения и распределение тепла медью PCB;
4. PCB-D и PCB-E должны пройти тепловой расчёт и испытание в закрытом объёме без контакта с корпусом;
5. расчётная проверка выполняется при верхней рабочей температуре среды электроники `+60 °C`;
6. если температурные пределы компонентов не выполняются, допускаются только: снижение потерь, изменение компонентов/топологии, увеличение площади/числа плат либо уменьшение разрешённой длительной нагрузки;
7. добавление теплового контакта с корпусом без нового решения владельца запрещено.

Это решение отменяет прежнее предварительное предложение о тепловом отводе PCB-D на корпус.

Статус:

```text
CLOSED_OWNER_THERMAL_CONSTRAINT; OPEN_THERMAL_CALC_TEST
```

## 7. Q-SYS-004 — short-limit timing

Владелец подтвердил:

```text
short-limit duration ≤1 s
minimum interval before repeated short event = 10 s
repeated events controlled by I²t / thermal accumulator
after 1 s continuous limit applies
```

Принято:

| Режим | Continuous | Short | Максимальная длительность short |
|---|---:|---:|---:|
| DUAL_PACK_RUN | 40 А | 44 А | 1 с |
| SINGLE_PACK_MODE | 20 А | 22 А | 1 с |

Повторный short-event программно не разрешается ранее 10 с и дополнительно зависит от состояния thermal accumulator. Аппаратная защита КЗ действует независимо.

Статус:

```text
CLOSED_PRELIMINARY_TIMING
```

## 9. Q-SYS-006 — critical/noncritical classification и load profiles

Владелец подтвердил предложенную политику первой версии.

Critical domain:

```text
5V_CRIT
3V3_CRIT
supervisor
fault manager
external communication
internal safety/control communication
journal/log retention
```

Некритические нагрузки для admission control:

```text
CH1…CH14
5V_OUT1…5V_OUT10
LED1…LED6
```

Для каждого назначенного внешнего устройства конфигурация хранит:

```text

```

Если профиль отсутствует:

- применяется консервативный максимальный профиль соответствующего канала;
- GUI показывает `LOAD_PROFILE_UNKNOWN`;
- команда не получает оптимистичную оценку тока.

Статус:

```text
CLOSED_OWNER_CLASSIFICATION
```

## 10. Q-SYS-007 — service override

Ответ владельца:

```text
Запрошено подробное объяснение назначения режима и предложенной политики.
```

Решение пока не принято.

До закрытия вопроса:

```text
SERVICE_OVERRIDE = DISABLED BY DEFAULT
```

Архитектура может резервировать программную возможность, но production firmware не должна позволять обход admission control без утверждённой политики.

Статус:

```text
OPEN_OWNER_DECISION_AFTER_EXPLANATION
```

## 11. Архитектурная проверка

Ответы V1.8 не изменяют:

- две основные АКБ 4S24P;
- параллельный DUAL_PACK_RUN;
- PACK_BUS;
- отсутствие K_MAIN;
- независимые BFE_1/BFE_2;
- моностабильные K_BAT1/K_BAT2;
- запрет automatic restart после BMS recovery;
- EXT_KILL, независимый от firmware/RS-485/CAN-FD;
- отдельные POWER_GND, SIGNAL_GND, ISO_GND и CHASSIS;
- внутренний CAN-FD и direct safety/fault lines;
- пассивный INTERCONNECT;
- отсутствие силовых токов через PCB-B.

Изменены только:

1. механический envelope электронной сборки;
2. способ крепления и извлечения;
3. thermal-path restriction;
4. численные параметры программного power-budget manager;
5. классификация внешних нагрузок.

## 12. Следующий этап

После принятия V1.8 разрешено:

1. вести компоновку в envelope `100 × 250 × 80 мм`;
2. закладывать многоуровневые платы со стойками и крепёжными отверстиями;
3. считать тепловые потери без использования корпуса как радиатора;
4. реализовывать short timer, cooldown и I²t accumulator;
5. реализовывать warning/block filtering и hysteresis;
6. формировать load-profile database;
7. продолжать component selection узлов с закрытыми электрическими входами.
