# PRECHARGE / BALANCE PATH

## 1. Назначение

Глобальный предзаряд `PACK_BUS → MAIN_INPUT_BUS` удалён вместе с `K_MAIN` и `MAIN_INPUT_BUS`.

Функция мягкого подключения реализуется отдельно в каждой батарейной ветви через ограниченный тракт:

```text
BATx → BALANCE_SWx → R_BALx → PACK_BUS
```

Этот тракт используется для:

1. мягкого подключения батарейной ветви к PACK_BUS;
2. ограничения пускового тока;
3. режима `DECK_BALANCE`;
4. диагностического контроля до включения основного силового тракта.

## 2. Структура одной ветви

```text
АКБ_x → K_BATx → СН-176А-12 → измерения →
    ├→ MAIN_SWx → PACK_BUS
    └→ BALANCE_SWx + R_BALx → PACK_BUS
```

## 3. Мягкое подключение ветви

1. Проверить наличие АКБ и состояние BMS.
2. Проверить отсутствие hard fault.
3. Замкнуть K_BATx.
4. Измерить BATx_V и PACK_BUS_V.
5. Включить BALANCE_SWx.
6. Контролировать BATx_I и PACK_BUS_V.
7. После достижения критерия мягкого подключения включить MAIN_SWx.
8. Отключить BALANCE_SWx, если он не нужен в текущем режиме.

## 4. DECK_BALANCE

В `DECK_BALANCE` ограниченный тракт остаётся активным для контролируемого межблочного тока. MCU контролирует:

1. BAT1_V и BAT2_V;
2. BAT1_I и BAT2_I;
3. температуры;
4. время операции;
5. SoC обоих блоков.

## 5. Защиты

1. ограничение максимального тока;
2. balance timeout;
3. защита R_BALx от перегрева;
4. запрет операции при BMS fault;
5. отключение при потере связи с критическим контроллером;
6. независимый аппаратный путь OFF.

## 6. Открытые расчёты

1. допустимый зарядный ток BMS;
2. номинальный ток DECK_BALANCE;
3. максимальный ток DECK_BALANCE;
4. R_BALx и его импульсная/длительная мощность;
5. допустимое время операции;
6. критерий завершения мягкого подключения;
7. критерий завершения DECK_BALANCE.
